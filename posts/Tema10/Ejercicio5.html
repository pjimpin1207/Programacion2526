Ejercicio 5: Consulta de Entidades (Búsqueda Directa y Lenguaje JPQL)

Enunciado

Una vez que hemos aprendido a persistir información en el disco duro con ObjectDB, el siguiente paso crítico es recuperarla de forma eficiente. En esta práctica aprenderemos a extraer entidades utilizando el acceso directo por clave primaria y el motor de consultas orientado a objetos JPQL (Java Persistence Query Language).

Objetivos del ejercicio:

  • Recuperar e imprimir el primer evento musical insertado (ID 10) utilizando el método de acceso directo por identificador único.
  • Recuperar e imprimir el segundo evento (ID 20) utilizando una Query Estática declarada mediante anotaciones en la propia entidad (@NamedQuery).
  • Recuperar de nuevo el evento mediante una Query Dinámica ensamblada en tiempo de ejecución.

Estructura de la Solución

1. La Entidad con Consulta Nombrada (EventoMusical)

Añadimos la anotación estructural @NamedQuery justo encima de la declaración de la clase. Esto permite al proveedor de persistencia precompilar y optimizar la consulta en tiempo de despliegue, mejorando el rendimiento y detectando errores de sintaxis de forma temprana.

package Tema10.Ejercicios.ejercicio5;

import Tema10.Ejercicios.ejercicio2.Artista;
import Tema10.Ejercicios.ejercicio2.GeneroMusical;

import javax.persistence.*;
import java.io.Serializable;
import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.ArrayList;
import java.util.List;

@Entity
// Definición de consulta estática reutilizable y parametrizada con :idParam
@NamedQuery(name = "buscarEventoPorIdEstatico", query = "SELECT e FROM EventoMusical e WHERE e.id = :idParam")
public class EventoMusical implements Serializable {
    private static final long serialVersionUID = 1L;

    @Id
    @GeneratedValue(generator = "Secuencia", strategy = GenerationType.SEQUENCE)
    @SequenceGenerator(name = "Secuencia", allocationSize = 10)
    private Long id;
    
    private String nombre;
    
    @Temporal(TemporalType.DATE)
    private LocalDate fecha;
    
    private BigDecimal recaudacion;
    private GeneroMusical genero;
    
    @ElementCollection
    private List<Artista> artistasConfirmados = new ArrayList<>();

    private transient int control;

    public EventoMusical() {
    }

    public EventoMusical(Long id, String nombre, LocalDate fecha, BigDecimal recaudacion, GeneroMusical genero, int control) {
        this.id = id;
        this.nombre = nombre;
        this.fecha = fecha;
        this.recaudacion = recaudacion;
        this.genero = genero;
        this.control = control;
    }

    // Getters y Setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }

    public LocalDate getFecha() { return fecha; }
    public void setFecha(LocalDate fecha) { this.fecha = fecha; }

    public BigDecimal getRecaudacion() { return recaudacion; }
    public void setRecaudacion(BigDecimal recaudacion) { this.recaudacion = recaudacion; }

    public GeneroMusical getGenero() { return genero; }
    public void setGenero(GeneroMusical genero) { this.genero = genero; }

    public List<Artista> getArtistasConfirmados() { return artistasConfirmados; }
    public void setArtistasConfirmados(List<Artista> artistasConfirmados) { this.artistasConfirmados = artistasConfirmados; }

    public int getControl() { return control; }
    public void setControl(int control) { this.control = control; }

    public void addArtista(Artista artista) {
        this.artistasConfirmados.add(artista);
    }

    @Override
    public String toString() {
        return "EventoMusical [ID (Incrementos de 10)=" + id + ", Nombre=" + nombre + ", Género=" + genero +
                ", Fecha=" + fecha + ", Recaudación=" + recaudacion + "€ | (Variable interna 'control'=" + control + ")]";
    }
}

2. Programa de Recuperación de Datos (Main)

No necesitamos iniciar transacciones (begin()/commit()) ya que las operaciones de lectura pura no modifican el estado de la base de datos. Empleamos el objeto tipado seguro TypedQuery para inyectar los parámetros de búsqueda de forma limpia.

package Tema10.Ejercicios.ejercicio5;

import javax.persistence.*;

public class Main {
    public static void main(String[] args) {
        // Conexión al motor de persistencia ObjectDB
        EntityManagerFactory emf = Persistence.createEntityManagerFactory("objectdb:$objectdb/db/eventos.odb");
        EntityManager em = emf.createEntityManager();

        // 1. MÉTODO DIRECTO: Extracción instantánea mediante clave primaria (ID 10L)
        System.out.println("--- 1. Recuperación Directa por ID ---");
        EventoMusical primerEvento = em.find(EventoMusical.class, 10L);
        System.out.println(primerEvento);
        System.out.println();

        // 2. QUERY ESTÁTICA: Invocación de la @NamedQuery declarada en la entidad (ID 20L)
        System.out.println("--- 2. Consulta JPQL Estática (NamedQuery) ---");
        TypedQuery<EventoMusical> queryEstatica = em.createNamedQuery("buscarEventoPorIdEstatico", EventoMusical.class);
        queryEstatica.setParameter("idParam", 20L); // Seteo seguro del parámetro indexado
        EventoMusical segundoEvento = queryEstatica.getSingleResult();
        System.out.println(segundoEvento);
        System.out.println();

        // 3. QUERY DINÁMICA: Ensamblado explícito de la cadena de consulta en tiempo de ejecución
        System.out.println("--- 3. Consulta JPQL Dinámica ---");
        TypedQuery<EventoMusical> queryDinamica = em.createQuery("SELECT e FROM EventoMusical e WHERE e.id = :idParam", EventoMusical.class);
        queryDinamica.setParameter("idParam", 10L);
        EventoMusical primerEvento2 = queryDinamica.getSingleResult();
        System.out.println(primerEvento2);

        // Liberación de canales de datos
        em.close();
        emf.close();
    }
}

Análisis de Estrategias de Consulta

Diferencias operativas en la recuperación de datos con JPA:

  • em.find(): Es el método de extracción más rápido y eficiente. Antes de realizar una consulta física al archivo de la base de datos, inspecciona el Contexto de Persistencia (caché de primer nivel). Si la entidad ya fue cargada previamente en la memoria RAM, la devuelve instantáneamente sin penalización de Entrada/Salida.
  • JPQL (Java Persistence Query Language): A diferencia de SQL tradicional (que realiza consultas sobre tablas y columnas físicas del esquema), JPQL es un lenguaje abstracto que realiza consultas **sobre las clases de objetos y sus atributos encapsulados** en Java. Observa que escribimos SELECT e FROM EventoMusical e haciendo referencia exacta al identificador de la clase, independientemente de cómo se llame el soporte de datos subyacente.
  • Consultas Estáticas vs. Dinámicas:
    • Las Queries Estáticas (@NamedQuery) se compilan una única vez cuando arranca la aplicación. Son más eficientes en ejecuciones masivas y aíslan el código de la lógica SQL/JPQL.
    • Las Queries Dinámicas (em.createQuery) se evalúan bajo demanda en tiempo de ejecución. Son indispensables cuando los criterios de filtrado dependen de elecciones del usuario que no se conocen de antemano (como marcar diferentes casillas de filtros dinámicos en una interfaz web).
  • Seguridad frente a Inyección: En ambas variantes de JPQL hacemos uso de .setParameter("idParam", valor). Jamás se deben concatenar variables directamente en la cadena de texto de la query (como "WHERE e.id = " + id), ya que el uso de parámetros tipados previene de forma absoluta los ataques de inyección maliciosa y asegura que el motor escape correctamente los caracteres de control.
