Ejercicio 10: Modificación Estructural de Tablas (Vaciado, Inserción y Auditoría)

Enunciado

En este ejercicio práctico realizaremos un script completo de inicialización y actualización de datos sobre la tabla de productos de la base de datos tienda. El programa debe automatizar de manera secuencial tres operaciones críticas de manipulación de datos (DML):

Acciones requeridas:

  • Limpiar de forma absoluta la tabla eliminando todos los registros antiguos existentes.
  • Realizar un volcado de inserción con un lote de tres nuevos productos (manzana, pera y uva) con sus respectivas propiedades y países de origen.
  • Actualizar de manera controlada el precio de uno de los productos (la pera) para fijarlo en un nuevo valor monetario.
  • Auditar en cada paso el número real de filas que han sido afectadas en el servidor de MariaDB.

Solución Completa

Para lanzar sentencias de manipulación destructiva o de inserción masiva en JDBC empleamos el método executeUpdate(), y para recuperar el número de filas modificadas por el motor recurrimos a la función getUpdateCount().

package Tema10.Ejercicios.ejercicio10;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;
import java.sql.Statement;

public class Ejercicio10 {
    public static void main(String[] args) throws Exception {
        try {
            // 1. Conexión segura con el servidor local relacional de MariaDB
            Connection con = DriverManager.getConnection("jdbc:mariadb://localhost:3306/tienda", "fran", "Fr4n");
            Statement stmt = con.createStatement();

            System.out.println("Conexión establecida");

            // 2. VACIADO SEGURO: Borrar todos los registros preexistentes en la tabla producto
            stmt.executeUpdate("DELETE FROM producto");
            // getUpdateCount() nos devuelve el número total de filas eliminadas en este paso
            System.out.println("Registros borrados en la limpieza: " + stmt.getUpdateCount());

            // 3. INSERCIÓN EN LOTE: Insertar los 3 nuevos registros solicitados por el enunciado
            stmt.executeUpdate("INSERT INTO producto VALUES ('1', 'manzana', 'Manzanas golden', 50, 'Francia')");
            stmt.executeUpdate("INSERT INTO producto VALUES ('2', 'pera', 'Peras conferencia', 25, 'Espana')");
            stmt.executeUpdate("INSERT INTO producto VALUES ('3', 'uva', 'Uvas groumet', 30, 'Espana')");
            // Muestra las filas afectadas por la última instrucción ejecutada en el canal
            System.out.println("Registros insertados en el lote: " + stmt.getUpdateCount());

            // 4. ACTUALIZACIÓN: Cambiar el precio del artículo 'pera' (ID 2) a 20 céntimos
            stmt.executeUpdate("UPDATE producto SET precio = 20 WHERE id = '2'");
            System.out.println("Registros actualizados de forma quirúrgica: " + stmt.getUpdateCount());

            // 5. LIBERACIÓN: Cierre absoluto de los canales descriptores y del socket de red
            stmt.close();
            con.close();

        } catch (SQLException e) {
            System.err.println("Ocurrió un error en la conexión o ejecución DML: " + e.getMessage());
        }
    }
}

Análisis Técnico de Operaciones DML en JDBC

Diferencias críticas entre consultas de lectura y sentencias de modificación:

  • El método executeUpdate(): A diferencia de las consultas de extracción (SELECT) que devuelven un cursor envuelto en un ResultSet tras invocar a executeQuery(), las sentencias de manipulación de datos o DML (INSERT, UPDATE, DELETE) se disparan de forma obligatoria mediante executeUpdate(). Este método procesa la orden y devuelve de forma directa un entero que representa la cantidad de filas afectadas en la tabla.
  • La propiedad getUpdateCount(): Es un método utilitario de la interfaz Statement. Almacena temporalmente el impacto de la última instrucción ejecutada con éxito en ese canal de comunicación. Es una excelente herramienta de auditoría para verificar en tiempo real si una cláusula restrictiva (como el WHERE id = '2') ha localizado e interactuado de forma correcta con el registro objetivo.
  • Estrategia de Transaccionalidad implícita: Por defecto, las conexiones JDBC operan bajo un modelo denominado Auto-commit activado en true. Esto significa que cada invocación a executeUpdate() consolida y graba el cambio de forma persistente en el disco instantáneamente. En entornos de producción con alta carga de datos, se suele desactivar mediante con.setAutoCommit(false) para agrupar todas las operaciones en un bloque atómico y consolidarlas al final con una única llamada a con.commit(), optimizando el rendimiento drásticamente.
