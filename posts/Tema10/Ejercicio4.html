Ejercicio 4: Persistencia Física en Base de Datos Orientada a Objetos (ObjectDB)

Enunciado

El objetivo de esta práctica es conectar nuestra aplicación Java con una base de datos real orientada a objetos (fichero con extensión .odb). Para ello, añadiremos los marcadores de metadatos de persistencia a nuestro POJO e inicializaremos el gestor de entidades para insertar los registros de manera persistente.

Requisitos del reto:

  • Marcar la clase EventoMusical como una entidad persistente gestionable.
  • Asegurar el mapeo correcto de la lista de objetos embebidos (artistas) utilizando las directivas de JPA.
  • Crear una base de datos local llamada eventos.odb.
  • Abrir una transacción segura, guardar físicamente dos eventos musicales completos con sus respectivos artistas y consolidar los cambios en el disco duro.

Estructura del Código

1. La Entidad Mapeada (EventoMusical)

Añadimos la anotación fundamental @Entity a nivel de clase. Además, para la lista de artistas (que al ser @Embeddable no tiene una tabla independiente), utilizamos la directiva @ElementCollection para indicarle al proveedor de persistencia que almacene la colección en una tabla subordinada de elementos.

package Tema10.Ejercicios.ejercicio4;

import Tema10.Ejercicios.ejercicio2.Artista;
import Tema10.Ejercicios.ejercicio2.GeneroMusical;

import javax.persistence.*;
import java.io.Serializable;
import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.ArrayList;
import java.util.List;

@Entity
public class EventoMusical implements Serializable {
    private static final long serialVersionUID = 1L;

    // Clave primaria secuencial indexada de 10 en 10
    @Id
    @GeneratedValue(generator = "Secuencia10en10", strategy = GenerationType.SEQUENCE)
    @SequenceGenerator(name = "Secuencia10en10", allocationSize = 10)
    private Long id;
    
    private String nombre;
    
    @Temporal(TemporalType.DATE)
    private LocalDate fecha;
    
    private BigDecimal recaudacion;
    private GeneroMusical genero;
    
    // ANOTACIÓN CLAVE: Define colecciones de elementos embebidos o tipos primitivos
    @ElementCollection
    private List<Artista> artistasConfirmados = new ArrayList<>();

    // Variable transitoria (Excluida por completo de la persistencia)
    private transient int control;

    // Constructor POJO por defecto
    public EventoMusical() {
    }

    // Constructor completo
    public EventoMusical(Long id, String nombre, LocalDate fecha, BigDecimal recaudacion, GeneroMusical genero, int control) {
        this.id = id;
        this.nombre = nombre;
        this.fecha = fecha;
        this.recaudacion = recaudacion;
        this.genero = genero;
        this.control = control;
    }

    // Getters y Setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }

    public LocalDate getFecha() { return fecha; }
    public void setFecha(LocalDate fecha) { this.fecha = fecha; }

    public BigDecimal getRecaudacion() { return recaudacion; }
    public void setRecaudacion(BigDecimal recaudacion) { this.recaudacion = recaudacion; }

    public GeneroMusical getGenero() { return genero; }
    public void setGenero(GeneroMusical genero) { this.genero = genero; }

    public List<Artista> getArtistasConfirmados() { return artistasConfirmados; }
    public void setArtistasConfirmados(List<Artista> artistasConfirmados) { this.artistasConfirmados = artistasConfirmados; }

    public int getControl() { return control; }
    public void setControl(int control) { this.control = control; }

    public void addArtista(Artista artista) {
        this.artistasConfirmados.add(artista);
    }

    @Override
    public String toString() {
        return "EventoMusical [ID (Incrementos de 10)=" + id + ", Nombre=" + nombre + ", Género=" + genero +
                ", Fecha=" + fecha + ", Recaudación=" + recaudacion + "€ | (Variable interna 'control'=" + control + ")]";
    }
}

2. Clase de Conexión y Gestión de Transacciones (Main)

Iniciamos el ciclo de vida de la persistencia. Construimos la factoría apuntando al fichero de base de datos local de ObjectDB (eventos.odb), abrimos una transacción aislando las operaciones críticas de base de datos con persist() y consolidamos con commit().

package Tema10.Ejercicios.ejercicio4;

import Tema10.Ejercicios.ejercicio2.Artista;
import Tema10.Ejercicios.ejercicio2.GeneroMusical;

import javax.persistence.EntityManager;
import javax.persistence.EntityManagerFactory;
import javax.persistence.Persistence;
import java.math.BigDecimal;
import java.time.LocalDate;

public class Main {
    public static void main(String[] args) {
        // 1. Instanciamos las entidades de prueba en memoria RAM
        EventoMusical festival1 = new EventoMusical(10L, "Resurrection Fest", LocalDate.of(2026, 6, 26), new BigDecimal("2450000.50"), GeneroMusical.LATINO, 1);
        festival1.addArtista(new Artista("Anuel", 1));
        festival1.addArtista(new Artista("Myke Towers", 1));

        EventoMusical festival2 = new EventoMusical(20L, "CocaCola Music Festival", LocalDate.of(2026, 7, 18), new BigDecimal("1890000.00"), GeneroMusical.POP, 2);
        festival2.addArtista(new Artista("Aitana", 1));

        // Auditamos visualmente los datos iniciales por consola
        System.out.println(festival1);
        System.out.println("Lista de artistas: " + festival1.getArtistasConfirmados() + "\n");
        System.out.println(festival2);
        System.out.println("Lista de artistas: " + festival2.getArtistasConfirmados() + "\n");

        // 2. Crear la factoría y abrir la conexión con la base de datos ObjectDB (.odb)
        EntityManagerFactory emf = Persistence.createEntityManagerFactory("objectdb:$objectdb/db/eventos.odb");
        EntityManager em = emf.createEntityManager();

        // 3. AISLAMIENTO SEGURO: Iniciar la transacción de datos
        em.getTransaction().begin();

        // 4. OPERACIONES DE PERSISTENCIA: Registramos las entidades en el mapa de control de JPA
        em.persist(festival1);
        em.persist(festival2);

        // 5. CONSOLIDACIÓN: Confirmar la transacción forzando la escritura física en disco
        em.getTransaction().commit();

        // 6. LIBERACIÓN: Cerrar los descriptores de conexión para evitar fugas de recursos (Memory Leaks)
        em.close();
        emf.close();

        System.out.println("¡Base de datos 'eventos.odb' generada e inserciones completadas con éxito!");
    }
}

Análisis de la Arquitectura JPA en Motores de Objetos

Mecánica del Ciclo de Vida de las Entidades:

  • @Entity: Esta anotación le indica al motor de persistencia que la clase tiene un reflejo directo en el almacenamiento permanente. En bases de datos relacionales tradicionales (como MariaDB) crearía una tabla, mientras que en una orientada a objetos (como ObjectDB) almacena directamente el grafo binario de la instancia.
  • @ElementCollection: Es obligatoria por contrato cuando mapeamos una lista (List) de objetos que no son entidades independientes (es decir, que no tienen su propio id ni anotación @Entity), como nuestra clase de utilidad Artista. Esto permite que los artistas se almacenen vinculados directamente al ciclo de vida del evento musical contenedor.
  • EntityManager y Transacciones: Cualquier operación de mutación, actualización o escritura (persist, remove, merge) exige obligatoriamente ser ejecutada dentro de un entorno acotado por begin() y commit(). Si el programa sufriese una caída o un corte de energía a mitad del proceso de persistencia, el motor ejecutaría un rollback automático, dejando la base de datos intacta y libre de datos corruptos o escrituras parciales.
