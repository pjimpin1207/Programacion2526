Ejercicio 3:

Enunciado

Continuamos dotando a nuestra capa de persistencia de herramientas avanzadas. En este ejercicio personalizaremos la estrategia de asignación automática de nuestra clave primaria e introduciremos variables operativas de control que deben quedar completamente aisladas de la base de datos.

Requisitos:

  • Configurar el campo identificador (ID) de la entidad para que utilice una secuencia cuyo incremento automático progrese de 10 en 10.
  • Introducir una variable de control numérica de tipo primitivo int llamada control.
  • Asegurar que dicha variable solo sirva para la lógica en memoria del programa y **no se refleje o mapee como columna** en el almacenamiento físico de la base de datos.

Solución Completa

Para configurar incrementos a medida en JPA empleamos las anotaciones @SequenceGenerator y @GeneratedValue. Para excluir la variable de la base de datos hacemos uso de la palabra clave nativa de Java transient.

1. Clase de Entidad (EventoMusical)

package Tema10.Ejercicios.ejercicio3;

import Tema10.Ejercicios.ejercicio2.Artista;
import Tema10.Ejercicios.ejercicio2.GeneroMusical;

import javax.persistence.*;
import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.ArrayList;
import java.util.List;

public class EventoMusical {

    // 1. Configuración de clave primaria con estrategia de incremento de 10 en 10
    @Id
    @GeneratedValue(generator = "Secuencia10en10", strategy = GenerationType.SEQUENCE)
    @SequenceGenerator(name = "Secuencia10en10", allocationSize = 10)
    private Long id;

    private String nombre;
    private LocalDate fecha;
    private BigDecimal recaudacion;
    private GeneroMusical genero;
    private List<Artista> artistasConfirmados = new ArrayList<>();

    // 2. Variable de control interna excluida explícitamente de la base de datos
    private transient int control;

    // Constructor vacío POJO
    public EventoMusical() {
    }

    // Constructor parametrizado completo
    public EventoMusical(Long id, String nombre, LocalDate fecha, BigDecimal recaudacion, GeneroMusical genero, int control) {
        this.id = id;
        this.nombre = nombre;
        this.fecha = fecha;
        this.recaudacion = recaudacion;
        this.genero = genero;
        this.control = control;
    }

    // Getters y Setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }

    public LocalDate getFecha() { return fecha; }
    public void setFecha(LocalDate fecha) { this.fecha = fecha; }

    public BigDecimal getRecaudacion() { return recaudacion; }
    public void setRecaudacion(BigDecimal recaudacion) { this.recaudacion = recaudacion; }

    public GeneroMusical getGenero() { return genero; }
    public void setGenero(GeneroMusical genero) { this.genero = genero; }

    public List<Artista> getArtistasConfirmados() { return artistasConfirmados; }
    public void setArtistasConfirmados(List<Artista> artistasConfirmados) { this.artistasConfirmados = artistasConfirmados; }

    public int getControl() { return control; }
    public void setControl(int control) { this.control = control; }

    public void addArtista(Artista artista) {
        this.artistasConfirmados.add(artista);
    }

    @Override
    public String toString() {
        return "EventoMusical [ID (Incrementos de 10)=" + id + ", Nombre=" + nombre + ", Género=" + genero +
                ", Fecha=" + fecha + ", Recaudación=" + recaudacion + "€ | (Variable interna 'control'=" + control + ")]";
    }
}

2. Clase Main

Instanciamos dos objetos simulando el salto indexado automático de las claves primarias (10 y 20) y asignamos valores operativos a la propiedad transitoria.

package Tema10.Ejercicios.ejercicio3;

import Tema10.Ejercicios.ejercicio2.Artista;
import Tema10.Ejercicios.ejercicio2.GeneroMusical;
import java.math.BigDecimal;
import java.time.LocalDate;

public class Main {
    public static void main(String[] args) {
        // Instancia del primer festival (ID de base 10)
        EventoMusical festival1 = new EventoMusical(10L, "Resurrection Fest", LocalDate.of(2026, 6, 26), new BigDecimal("2450000.50"), GeneroMusical.LATINO, 1);
        festival1.addArtista(new Artista("Anuel", 1));
        festival1.addArtista(new Artista("Myke Towers", 1));

        // Instancia del segundo festival (ID de salto a 20)
        EventoMusical festival2 = new EventoMusical(20L, "CocaCola Music Festival", LocalDate.of(2026, 7, 18), new BigDecimal("1890000.00"), GeneroMusical.POP, 2);
        festival2.addArtista(new Artista("Aitana", 1));

        // Volcado de auditoría por consola para contrastar los incrementos
        System.out.println(festival1);
        System.out.println("Lista de artistas: " + festival1.getArtistasConfirmados());
        System.out.println();

        System.out.println(festival2);
        System.out.println("Lista de artistas: " + festival2.getArtistasConfirmados());
    }
}

Análisis de Conceptos Avanzados

1. Generadores de Secuencia Personalizados

Por defecto, estrategias como GenerationType.IDENTITY incrementan el ID de uno en uno confiando en la columna autoincremental de motores como MySQL. Al utilizar GenerationType.SEQUENCE, vinculamos un motor de secuencias independiente (típico en PostgreSQL u Oracle). El atributo clave es allocationSize = 10, que determina cuántos valores reserva el framework en memoria por cada consulta física a la base de datos, forzando la progresión de 10 en 10 requerida.

2. El Modificador transient (Exclusión)

La palabra reservada transient es una directiva de seguridad y arquitectura que cumple una doble función en Java:

  • En serialización nativa: Indica al flujo ObjectOutputStream que ignore el atributo al congelar el objeto, por lo que su valor se destruye al volcar los bytes al disco duro.
  • En persistencia ORM (JPA/Hibernate): Actúa de la misma forma que la anotación @Transient, indicando de forma explícita al mapeador que no cree una columna para esta variable en la tabla relacional, permitiéndonos disponer de banderas lógicas en memoria RAM sin sobrecargar nuestro esquema físico de base de datos.
