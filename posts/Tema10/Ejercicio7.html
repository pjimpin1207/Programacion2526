Ejercicio 7: Clonación y Eliminación de Entidades (Ciclo de Vida y Sentencias DELETE)

Enunciado

En este ejercicio cerramos las operaciones CRUD fundamentales de JPA explorando la eliminación de registros. El reto exige realizar un borrado de seguridad copiando previamente un elemento original a un registro nuevo, para luego proceder a limpiar datos concretos por clave primaria y mediante consultas masivas de tipo DELETE en JPQL.

Misiones requeridas:

  • Buscar el primer evento (ID 10) y generar una copia idéntica con un nuevo identificador autoincremental antes de destruirlo.
  • Eliminar el primer evento original utilizando su identificador mediante métodos directos del gestor de persistencia.
  • Eliminar el segundo evento (ID 20) utilizando una Query JPQL Dinámica de tipo DELETE.
  • Eliminar de nuevo de forma controlada el segundo evento valiéndose de una Query JPQL Estática declarada mediante @NamedQuery.

Solución Completa

1. La Entidad con Consultas de Eliminación Nombradas (EventoMusical)

Declaramos de manera estática tanto la query de consulta como la de eliminación dentro de la anotación multi-consulta @NamedQueries en el encabezado de nuestra clase de entidad.

package Tema10.Ejercicios.ejercicio7;

import Tema10.Ejercicios.ejercicio2.Artista;
import Tema10.Ejercicios.ejercicio2.GeneroMusical;

import javax.persistence.*;
import java.io.Serializable;
import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.ArrayList;
import java.util.List;

@Entity
@NamedQueries({
        @NamedQuery(name = "buscarEventoPorIdEstatico", query = "SELECT e FROM EventoMusical e WHERE e.id = :idParam"),
        @NamedQuery(name = "borrarEventoEstatico", query = "DELETE FROM EventoMusical e WHERE e.id = :idParam")
})
public class EventoMusical implements Serializable {
    private static final long serialVersionUID = 1L;

    @Id
    @GeneratedValue(generator = "SecuenciaEj7", strategy = GenerationType.SEQUENCE)
    @SequenceGenerator(name = "SecuenciaEj7", allocationSize = 10)
    private Long id;
    
    private String nombre;
    @Temporal(TemporalType.DATE)
    private LocalDate fecha;
    private BigDecimal recaudacion;
    private GeneroMusical genero;
    
    @ElementCollection
    private List<Artista> artistasConfirmados = new ArrayList<>();

    private transient int control;

    public EventoMusical() {
    }

    public EventoMusical(Long id, String nombre, LocalDate fecha, BigDecimal recaudacion, GeneroMusical genero, int control) {
        this.id = id;
        this.nombre = nombre;
        this.fecha = fecha;
        this.recaudacion = recaudacion;
        this.genero = genero;
        this.control = control;
    }

    // Getters y Setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }

    public LocalDate getFecha() { return fecha; }
    public void setFecha(LocalDate fecha) { this.fecha = fecha; }

    public BigDecimal getRecaudacion() { return recaudacion; }
    public void setRecaudacion(BigDecimal recaudacion) { this.recaudacion = recaudacion; }

    public GeneroMusical getGenero() { return genero; }
    public void setGenero(GeneroMusical genero) { this.genero = genero; }

    public List<Artista> getArtistasConfirmados() { return artistasConfirmados; }
    public void setArtistasConfirmados(List<Artista> artistasConfirmados) { this.artistasConfirmados = artistasConfirmados; }

    public int getControl() { return control; }
    public void setControl(int control) { this.control = control; }

    public void addArtista(Artista artista) {
        this.artistasConfirmados.add(artista);
    }

    @Override
    public String toString() {
        return "EventoMusical [ID (Incrementos de 10)=" + id + ", Nombre=" + nombre + ", Género=" + genero +
                ", Fecha=" + fecha + ", Recaudación=" + recaudacion + "€ | (Variable interna 'control'=" + control + ")]";
    }
}

2. Programa de Procesamiento Aislado (BorrarEventos)

Al igual que ocurre con las modificaciones estipuladas en la lección anterior, los borrados estructurales alteran el mapa físico de la base de datos y exigen de manera estricta ejecutarse dentro de transacciones acotadas.

package Tema10.Ejercicios.ejercicio7;

import javax.persistence.EntityManager;
import javax.persistence.EntityManagerFactory;
import javax.persistence.Persistence;
import javax.persistence.Query;
import java.util.ArrayList;

public class BorrarEventos {
    public static void main(String[] args) {
        // Inicializar la factoría apuntando al fichero local eventos.odb
        EntityManagerFactory emf = Persistence.createEntityManagerFactory("objectdb:$objectdb/db/eventos.odb");
        EntityManager em = emf.createEntityManager();

        // FASE 1: Buscar, Duplicar como Backup y Eliminar mediante Identificador
        em.getTransaction().begin();

        // Extraemos la entidad en estado Gestionado (Managed)
        EventoMusical primerEvento = em.find(EventoMusical.class, 10L);

        // Instanciamos un nuevo objeto pasando 'null' en el ID para que la secuencia le asigne uno nuevo
        EventoMusical copiaEvento = new EventoMusical(
                null,
                primerEvento.getNombre() + " (Copia)",
                primerEvento.getFecha(),
                primerEvento.getRecaudacion(),
                primerEvento.getGenero(),
                primerEvento.getControl()
        );
        // Clonamos la lista de artistas confirmados en una nueva instancia de colección para romper referencias
        copiaEvento.setArtistasConfirmados(new ArrayList<>(primerEvento.getArtistasConfirmados()));

        // Guardamos la copia de respaldo en el motor
        em.persist(copiaEvento);

        // Eliminamos de forma física la entidad original a través de su puntero de control
        em.remove(primerEvento);

        em.getTransaction().commit();
        System.out.println("Primer evento copiado y eliminado mediante identificador con éxito.");

        // FASE 2: Eliminación masiva mediante JPQL Dinámico (ID 20L)
        em.getTransaction().begin();
        Query queryDinamica = em.createQuery("DELETE FROM EventoMusical e WHERE e.id = 20");
        int eliminadosDin = queryDinamica.executeUpdate(); // Devuelve el número de registros afectados
        em.getTransaction().commit();
        System.out.println("Registros eliminados con JPQL Dinámica: " + eliminadosDin);

        // FASE 3: Eliminación masiva mediante JPQL Estático (NamedQuery)
        em.getTransaction().begin();
        Query queryEstatica = em.createNamedQuery("borrarEventoEstatico");
        queryEstatica.setParameter("idParam", 20L);
        int eliminadosEst = queryEstatica.executeUpdate();
        em.getTransaction().commit();
        System.out.println("Registros eliminados con JPQL Estática: " + eliminadosEst);

        // Cierre y liberación de flujos de comunicación con la base de datos
        em.close();
        emf.close();
    }
}

Análisis de la Mecánica de Eliminación en JPA

Claves de diseño en la destrucción de información persistente:

  • Desvinculación en la clonación (ID null): Al duplicar un objeto persistente, si pasáramos el ID original (10L) a la copia, JPA interpretaría que queremos actualizar el registro existente en lugar de crear uno nuevo. Al asignarle null, la anotación @GeneratedValue entra en juego de forma automática solicitando un nuevo valor a la secuencia, logrando el duplicado de seguridad de forma limpia.
  • El método em.remove(): Este método exige de forma obligatoria que el argumento proporcionado sea una entidad en estado **Managed (Gestionada)**. Por este motivo, siempre realizamos un paso previo de búsqueda con em.find(). Invocar a em.remove() sobre un objeto construido manualmente con new producirá un fallo de tipo IllegalArgumentException (Removing a detached instance).
  • Eliminaciones en lote (Bulk Delete): Al igual que con las sentencias UPDATE estudiadas en la lección anterior, los comandos DELETE FROM... de JPQL se ejecutan directamente en el motor de base de datos mediante **executeUpdate()**. Esta vía es extremadamente eficiente si se desea realizar una limpieza masiva por criterios lógicos (por ejemplo, borrar todos los eventos pasados de fecha) sin incurrir en el costoso proceso de cargar miles de objetos a la memoria RAM de la aplicación uno a uno.
