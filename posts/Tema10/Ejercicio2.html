Ejercicio 2: Preparando un POJO para la Persistencia en Base de Datos

Enunciado

A partir del POJO básico del ejercicio anterior, vamos a refinar y dotar a nuestro modelo de dominio de las restricciones estructurales necesarias para que pueda ser almacenado y mapeado de manera directa en una base de datos relacional (JPA) o de objetos (ObjectDB).

Requisitos y Ampliaciones:

  • Implementar el contrato de serialización e incluir un identificador (ID) único de tipo Long que actúe como clave primaria en la base de datos.
  • Restringir el campo del género musical mediante el uso explícito de un **Tipo Enumerado (Enum)**.
  • Gestionar una lista interna de artistas confirmados utilizando una clase puente independiente de soporte (Artista) marcada como **objeto embebido**, evitando que genere una tabla huérfana en la persistencia.

Estructura del Código

Dividimos el diseño de la solución en cuatro archivos fuente para mantener un desacoplamiento de componentes limpio y escalable.

1. Tipo Enumerado (GeneroMusical)

package Tema10.Ejercicios.ejercicio2;

public enum GeneroMusical {
    ROCK, POP, RAP, TRAP, TECHNO, INDIE, RECCAE, LATINO
}

2. Clase de Componente Embebido (Artista)

Utilizamos la anotación estándar @Embeddable para indicar a los motores de base de datos que los campos de esta clase deben integrarse directamente como columnas embebidas dentro de la entidad contenedora.

package Tema10.Ejercicios.ejercicio2;

import javax.persistence.Embeddable;
import java.io.Serializable;

@Embeddable
public class Artista implements Serializable {
    private static final long serialVersionUID = 1L;

    private String nombreArtistico;
    private int numeroIntegrantes;

    // Constructor vacío obligatorio por convenio JavaBean / JPA
    public Artista() {
        super();
    }

    public Artista(String nombreArtistico, int numeroIntegrantes) {
        this.nombreArtistico = nombreArtistico;
        this.numeroIntegrantes = numeroIntegrantes;
    }

    // Getters y Setters
    public String getNombreArtistico() { return nombreArtistico; }
    public void setNombreArtistico(String nombreArtistico) { this.nombreArtistico = nombreArtistico; }

    public int getNumeroIntegrantes() { return numeroIntegrantes; }
    public void setNumeroIntegrantes(int numeroIntegrantes) { this.numeroIntegrantes = numeroIntegrantes; }

    @Override
    public String toString() {
        return nombreArtistico + " (" + numeroIntegrantes + " miembros)";
    }
}

3. Clase de Entidad Principal (EventoMusical)

package Tema10.Ejercicios.ejercicio2;

import java.io.Serializable;
import java.math.BigDecimal;
import java.time.LocalDate;
import java.util.ArrayList;
import java.util.List;

public class EventoMusical implements Serializable {
    private static final long serialVersionUID = 1L;

    // Clave primaria esencial indexada para identificar el registro en la BD
    private Long id;

    private String nombre;
    private LocalDate fecha;
    private BigDecimal recaudacion;

    // Acoplamos la enumeración restrictiva de tipos de género
    private GeneroMusical genero;

    // Colección de elementos embebidos de tipo Artista
    private List<Artista> artistasConfirmados = new ArrayList<>();

    public EventoMusical() {
    }

    public EventoMusical(Long id, String nombre, LocalDate fecha, BigDecimal recaudacion, GeneroMusical genero) {
        this.id = id;
        this.nombre = nombre;
        this.fecha = fecha;
        this.recaudacion = recaudacion;
        this.genero = genero;
    }

    // Getters y Setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }

    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }

    public LocalDate getFecha() { return fecha; }
    public void setFecha(LocalDate fecha) { this.fecha = fecha; }

    public BigDecimal getRecaudacion() { return recaudacion; }
    public void setRecaudacion(BigDecimal recaudacion) { this.recaudacion = recaudacion; }

    public GeneroMusical getGenero() { return genero; }
    public void setGenero(GeneroMusical genero) { this.genero = genero; }

    public List<Artista> getArtistasConfirmados() { return artistasConfirmados; }
    public void setArtistasConfirmados(List<Artista> artistasConfirmados) { this.artistasConfirmados = artistasConfirmados; }

    // Método utilitario helper para añadir artistas de forma controlada a la colección
    public void addArtista(Artista artista) {
        this.artistasConfirmados.add(artista);
    }

    @Override
    public String toString() {
        return "EventoMusical [ID=" + id + ", Nombre=" + nombre + ", Género=" + genero +
                ", Fecha=" + fecha + ", Recaudación=" + recaudacion + "€\n Artistas=" + artistasConfirmados + "]";
    }
}

4. Clase Main

package Tema10.Ejercicios.ejercicio2;

import java.math.BigDecimal;
import java.time.LocalDate;

public class Main {
    public static void main(String[] args) {
        // Instanciamos la entidad pasándole un tipo Long para el ID (sufijo L)
        EventoMusical festival = new EventoMusical(1L, "Puro Latino", LocalDate.of(2026, 6, 28), new BigDecimal("3450000.00"), GeneroMusical.LATINO);

        // Añadimos las instancias de los artistas mediante el método helper
        festival.addArtista(new Artista("Anuel", 8));
        festival.addArtista(new Artista("Myke Towers", 9));

        // Auditamos el resultado final estructurado en consola
        System.out.println(festival);
    }
}

Conceptos de Persistencia Clave

  • El Wrapper Long para IDs: Se prefiere la clase envoltorio Long en lugar del tipo primitivo long para representar la clave primaria debido a que permite el uso de valores nulos (null). Un ID nulo es la señal universal que usan los frameworks ORM (como Hibernate) para detectar que un objeto es una entidad nueva que aún no ha sido persistida en disco.
  • Garantía Estructural con Enums: Declarar el género como GeneroMusical en lugar de un simple String bloquea en tiempo de compilación la inserción de texto inválido o erratas ortográficas, asegurando que la base de datos solo almacene estados de dominios válidos de manera unificada.
  • Concepto de Objeto Embebido (@Embeddable): Según el enunciado, cada artista no debe suponer una entidad independiente. Al marcar la clase como embebible, le indicamos al motor que los campos nombreArtistico y numeroIntegrantes se inyectarán de forma integrada dentro de la tabla del propio evento, ahorrando costosas operaciones de cruzado de tablas (JOINS) y simplificando el mapa relacional del esquema de la base de datos.
