Ejercicio 9: Extracción y Volcado de Datos con JDBC (El uso de ResultSet)

Enunciado

Tras consolidar la conexión básica con el motor relacional de MariaDB, el siguiente paso es interactuar con las tablas para extraer información. El ejercicio requiere insertar previamente tres registros en la tabla de productos mediante la CLI (línea de comandos) de MariaDB y, posteriormente, implementar un programa en Java que recupere dicha lista ordenada por coste de manera descendente.

Objetivos de la práctica:

  • Efectuar inserciones manuales de prueba mediante sentencias INSERT INTO desde la terminal de MariaDB.
  • Diseñar una consulta de selección estructurada en Java utilizando la cláusula de ordenación nativa ORDER BY precio DESC.
  • Recorrer dinámicamente el cursor de datos devuelto por el servidor y volcar las propiedades de cada producto por la consola del sistema.

Solución Completa

Para recuperar registros de una base de datos relacional empleamos el método executeQuery() de la interfaz Statement, el cual captura la información estructurándola dentro de un objeto contenedor especial llamado ResultSet.

package Tema10.Ejercicios.ejercicio9;

import java.sql.*;

public class Ejercicio9 {
    public static void main(String[] args) {
        try {
            // 1. Establecer la conexión con el servidor local de MariaDB
            Connection con = DriverManager.getConnection("jdbc:mariadb://localhost:3306/tienda", "fran", "Fr4n");
            Statement stmt = con.createStatement();

            System.out.println("Conexión establecida");

            // 2. Definición de la consulta SQL aplicando ordenación de mayor a menor precio
            String query = "SELECT * FROM producto ORDER BY precio DESC";
            
            // 3. Ejecución de la query: Almacenamos el puntero de resultados en el ResultSet
            ResultSet resultados = stmt.executeQuery(query);

            System.out.println("\n--- Productos ordenados por precio (Mayor a Menor) ---");
            
            // 4. Bucle de extracción: resultados.next() avanza el cursor al siguiente registro disponible
            while (resultados.next()) {
                // Recuperamos las columnas del registro actual mediante su nombre identificador
                String id = resultados.getString("id");
                String nombre = resultados.getString("nombre");
                int precio = resultados.getInt("precio"); // El precio se gestiona en céntimos para evitar decimales
                String pais = resultados.getString("pais");

                System.out.println("ID: " + id + " | Nombre: " + nombre + " | Precio: " + precio + " céntimos | País: " + pais);
            }

            // 5. LIBERACIÓN DE RECURSOS: Cerramos el ResultSet, el Statement y la Connection de forma escalonada
            resultados.close();
            stmt.close();
            con.close();

        } catch (SQLException e) {
            System.err.println("No se pudo conectar o procesar la consulta SQL: " + e.getMessage());
        }
    }
}

Análisis Técnico del Procesamiento SQL con JDBC

Comprendiendo el funcionamiento de ResultSet y Cursores:

  • Mecánica de executeQuery(): A diferencia de operaciones de modificación (como INSERT, UPDATE o DELETE) que se disparan mediante executeUpdate(), las operaciones de extracción pura de lectura (SELECT) exigen el uso estricto de executeQuery(), el cual se encarga de devolver la tabla de resultados estructurada.
  • El funcionamiento del Cursor (.next()): Un objeto ResultSet no vuelca todos los datos de golpe en un array de la memoria RAM. En su lugar, mantiene una conexión abierta con el servidor controlando un cursor lógico. Inicialmente, este cursor apunta justo antes de la primera fila. Al invocar a resultados.next() dentro del bucle condicional, ocurren dos cosas:
    1. El cursor se desplaza físicamente hacia el siguiente registro de la tabla.
    2. Devuelve un valor booleano true si la celda contiene un registro válido, o false de manera automática cuando alcanza el final de las filas (EOF), rompiendo el bucle de forma limpia.
  • Métodos Getter Tipados: Para extraer los datos de las columnas de la fila actual, hacemos uso de métodos específicos adaptados al tipo de datos declarado en el esquema de la tabla de la base de datos (como getString("nombre") o getInt("precio")), pasando como argumento el nombre exacto de la columna del catálogo SQL.
  • El orden de Cierre de Descriptores: Las conexiones JDBC son flujos de hardware pesados que consumen sockets del sistema operativo. Es imperativo cerrarlos en orden inverso a su apertura (primero los resultados, luego los canales de sentencias y por último el puente de red), asegurando la liberación absoluta de recursos.
