Resumen de la Unidad 8: Colecciones y TAD

Esta unidad explora cómo Java permite modelar problemas complejos mediante el uso de Tipos Abstractos de Datos (TAD) y el potente Java Collections Framework (JCF).

1. Introducción a los TAD

Un TAD es un modelo definido para representar tipos abstractos de datos (como una pila o un conjunto)[cite: 1]. Se caracteriza por definirse mediante una interfaz y contener tanto los datos como las operaciones (POO).

  • Ejemplos: Listas, Pilas (LIFO), Colas (FIFO), Conjuntos, Diccionarios y Grafos.

2. Java Collections Framework (JCF)

Ubicado en el paquete java.util.*, este framework proporciona una arquitectura común para manejar grupos de objetos.

2.1 Listas (Interface List)

Colecciones secuenciales que permiten duplicados y mantienen el orden de inserción.

  • ArrayList: Arrays dinámicos ideales para lectura frecuente (acceso aleatorio).
  • LinkedList: Lista doblemente enlazada, excelente para inserciones y eliminaciones constantes.

2.2 Conjuntos (Interface Set)

Estructuras que no permiten elementos duplicados.

  • HashSet: El más rápido, pero sin orden garantizado (caos total).
  • LinkedHashSet: Mantiene el orden de llegada.
  • TreeSet: Ordena los elementos de forma natural (alfabética o numérica) usando un árbol binario.

2.3 Mapas (Interface Map)

Representan diccionarios de pares Clave - Valor. Las claves son únicas, pero los valores pueden repetirse.

  • HashMap: Rápido y sin orden específico.
  • TreeMap: Ordena las claves de forma natural.
  • LinkedHashMap: Respeta el orden de inserción.

3. Conceptos Avanzados

Wrappers y Genéricos

Las colecciones solo admiten objetos, por lo que usamos Wrappers (como Integer o Double) para envolver tipos primitivos. Los Tipos Genéricos (<T>) aportan seguridad de tipos y eliminan la necesidad de realizar casts manuales.

Comparadores

  • Comparable: Interfaz para establecer el orden natural de una clase (compareTo).
  • Comparator: Permite definir órdenes alternativos mediante clases externas (compare).

Iteradores

El Iterator es el TAD de Java diseñado específicamente para recorrer colecciones de forma segura, permitiendo eliminar elementos durante el recorrido sin errores estructurales.

4. Clase de utilidad Collections

A diferencia de la interfaz Collection, Collections es una clase utilitaria con métodos estáticos para operar sobre las listas:

  • sort(): Ordenar elementos.
  • shuffle(): Barajar la lista aleatoriamente.
  • max() / min(): Encontrar valores extremos.
  • frequency(): Contar repeticiones de un objeto.
Nota Importante: Programar siempre orientado a interfaces (ej. List lista = new ArrayList()) es una mejor práctica que usar las implementaciones directamente.
