Ejercicio 11: Iteradores - Gestión de un Club de Lectura

Enunciado

El objetivo de este ejercicio es gestionar una biblioteca de libros utilizando un HashSet para evitar duplicados y practicar las diferentes formas de recorrer una colección, especialmente mediante el uso de Iteradores.

Requisitos:

  • Almacenar nombres de libros en un HashSet<String>.
  • Permitir al usuario agregar libros dinámicamente.
  • Recorrer y mostrar el contenido utilizando un Iterator.
  • Reflexionar sobre el funcionamiento interno de los conjuntos y los riesgos de modificar colecciones mientras se recorren.

Solución

Dividimos la lógica en una clase de servicio ClubDeLectura que gestiona la colección y una clase principal para la interacción con el usuario.

1. Clase ClubDeLectura

package Tema8.Ejercicios.ejercicio11;

import java.util.HashSet;
import java.util.Iterator;
import java.util.Set;

public class ClubDeLectura {
    private Set<String> biblioteca;

    public ClubDeLectura() {
        // Inicializamos el conjunto para evitar duplicados
        biblioteca = new HashSet<String>();
    }

    public void AgregarLibro(String libro) {
        if (biblioteca.add(libro)) {
            System.out.println(libro + " ha sido añadido correctamente.");
        } else {
            System.out.println("Error: el libro '" + libro + "' ya está en la biblioteca.");
        }
    }

    public void MostrarBiblioteca() {
        System.out.println("--- Libros de la biblioteca ---");

        // Uso de Iterator para recorrer el conjunto
        Iterator<String> it = biblioteca.iterator();
        while (it.hasNext()) {
            System.out.println("- " + it.next());
        }
    }
}

2. Clase Principal (Main)

package Tema8.Ejercicios.ejercicio11;

import java.util.Scanner;

public class Ejercicio11 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        ClubDeLectura club = new ClubDeLectura();

        // Proceso de entrada de datos
        for (int i = 0; i < 3; i++) {
            System.out.print("Introduce el nombre de un libro: ");
            club.AgregarLibro(sc.nextLine());
        }

        // Mostrar el resultado final
        System.out.println();
        club.MostrarBiblioteca();
        
        sc.close();
    }
}

Reflexión Teórica

1. ¿Por qué HashSet no garantiza un orden específico?
Porque utiliza una tabla Hash para almacenar los elementos. El orden depende del valor del hashcode de cada objeto y de la capacidad de la tabla, no del orden en el que fueron insertados.

2. ¿Cuándo es preferible usar un Iterator en lugar de un for-each?
El Iterator es imprescindible cuando necesitamos eliminar elementos de la colección mientras la estamos recorriendo de forma segura mediante el método it.remove().

3. ¿Qué pasaría si intentamos modificar el Set con un for-each?
Java lanzaría una excepción de tipo ConcurrentModificationException. El bucle for-each no permite alterar la estructura de la colección (añadir o borrar) mientras se itera sobre ella.
