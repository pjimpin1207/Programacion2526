Ejercicio 10: Comparadores - Flexibilidad en la Ordenación

Enunciado

En el ejercicio anterior definimos el orden natural de una Persona (por nombre). Sin embargo, en aplicaciones reales solemos necesitar ordenar los mismos datos de diferentes formas según el contexto. Aquí es donde entra en juego la interfaz Comparator.

Requisitos:

  • Crear un comparador externo para ordenar por edad.
  • Crear otro comparador para ordenar por apellidos.
  • Implementar un comparador combinado: Apellidos > Nombre > Edad.
  • Ampliación: Gestionar valores nulos para que las personas sin apellido aparezcan al principio de la lista.

Solución: Implementación de Comparadores

A diferencia de Comparable, los comparadores se definen en clases independientes que implementan Comparator<T>, lo que permite tener infinitas formas de ordenar sin tocar el código de la clase Persona.

1. Clases Comparadoras

package Tema8.Ejercicios.ejercicio10;

import Tema8.Ejercicios.ejercicio9.Persona;
import java.util.Comparator;

// Estrategia 1: Comparar solo por edad
class ComparadorEdad implements Comparator<Persona> {
    @Override
    public int compare(Persona p1, Persona p2) {
        return Integer.compare(p1.getEdad(), p2.getEdad());
    }
}

// Estrategia 2: Comparador Combinado con manejo de nulos (Ampliación)
class ComparadorCombinado implements Comparator<Persona> {
    @Override
    public int compare(Persona p1, Persona p2) {
        // Lógica para apellidos nulos: mandarlos al principio (-1)
        if (p1.getApellidos() == null && p2.getApellidos() != null) return -1;
        if (p1.getApellidos() != null && p2.getApellidos() == null) return 1;

        int res = 0;
        // Si ambos tienen apellido, comparamos alfabéticamente
        if (p1.getApellidos() != null && p2.getApellidos() != null) {
            res = p1.getApellidos().compareTo(p2.getApellidos());
        }

        // Si el apellido es idéntico o ambos nulos, pasamos al Nombre
        if (res == 0) {
            res = p1.getNombre().compareTo(p2.getNombre());
        }

        // Si el nombre también coincide, decidimos por Edad
        if (res == 0) {
            res = Integer.compare(p1.getEdad(), p2.getEdad());
        }

        return res;
    }
}

2. Método Main

package Tema8.Ejercicios.ejercicio10;

import Tema8.Ejercicios.ejercicio9.Persona;
import java.util.*;

public class Main {
    public static void main(String[] args) {
        List<Persona> lista = new ArrayList<>();
        lista.add(new Persona("Fran", "Gomez", 30));
        lista.add(new Persona("Ana", "Zaragoza", 25));
        lista.add(new Persona("Luis", null, 40)); // Persona con apellido nulo

        // Ordenación por edad usando Collections.sort
        System.out.println("--- Orden por Edad ---");
        Collections.sort(lista, new ComparadorEdad());
        System.out.println(lista);

        // Ordenación combinada (Apellidos > Nombre > Edad)
        System.out.println("\n--- Orden Combinado (Nulos al principio) ---");
        Collections.sort(lista, new ComparadorCombinado());
        System.out.println(lista);
    }
}

Diferencias Fundamentales

Característica Comparable Comparator
Ubicación Dentro de la clase (this). Clase externa o anónima.
Flexibilidad Un solo criterio (Natural). Múltiples criterios personalizados.
Método compareTo(obj) compare(obj1, obj2)
