Ejercicio 14: Comparativa de Mapas - El Ranking de Competición

Enunciado

¿Cuándo debemos usar un mapa u otro? En este ejercicio registraremos participantes de una competición de programación y compararemos cómo almacenan los datos un HashMap, un LinkedHashMap y un TreeMap.

Instrucciones:

  • Crear las estructuras: Instanciar los tres tipos de mapas.
  • Carga de datos: Insertar los mismos participantes ("Zoe", "Bernat", "Ana", "Mario") en el mismo orden.
  • Recorrido con Iteradores: Implementar un método genérico que use Iterator sobre el entrySet del mapa para mostrar los resultados.

Solución

A continuación, el código que demuestra cómo cada implementación organiza las llaves (nombres de participantes) de manera distinta según su lógica interna.

package Tema8.Ejercicios.ejercicio14;

import java.util.*;

public class Ejercicio14 {
    public static void main(String[] args) {
        // 1. Creación de las tres estructuras diferentes
        Map<String, Integer> aleatorio = new HashMap<>();
        Map<String, Integer> insercion = new LinkedHashMap<>();
        Map<String, Integer> alfabetico = new TreeMap<>();

        // Datos para la competición
        String[] nombres = {"Zoe", "Bernat", "Ana", "Mario"};
        int[] notas = {85, 92, 78, 88};

        // 2. Carga idéntica de datos en todos los mapas
        for (int i = 0; i < nombres.length; i++) {
            aleatorio.put(nombres[i], notas[i]);
            insercion.put(nombres[i], notas[i]);
            alfabetico.put(nombres[i], notas[i]);
        }

        // 3. Ejecución del método de visualización
        mostrarMapa(aleatorio, "HASHMAP (Sin orden garantizado)");
        mostrarMapa(insercion, "LINKEDHASHMAP (Orden de inserción)");
        mostrarMapa(alfabetico, "TREEMAP (Orden natural/alfabético)");
    }

    // Método genérico para recorrer cualquier Map usando Iterator sobre entrySet
    public static void mostrarMapa(Map<String, Integer> mapa, String tipo) {
        System.out.println("\n--- " + tipo + " ---");

        // Obtenemos el iterador del conjunto de entradas (clave-valor)
        Iterator<Map.Entry<String, Integer>> it = mapa.entrySet().iterator();

        while (it.hasNext()) {
            Map.Entry<String, Integer> entrada = it.next();
            System.out.println("Participante: " + entrada.getKey() + " | Puntos: " + entrada.getValue());
        }
    }
}

Cuestionario de Observación

1. ¿Qué ha pasado en el HashMap?
Los nombres aparecen en un orden aleatorio. Esto ocurre porque organiza los elementos según el hashCode de la clave para maximizar la velocidad.

2. ¿Qué ha pasado en el LinkedHashMap?
Se ha respetado fielmente el orden de llegada. Esta estructura utiliza una lista doblemente enlazada para "recordar" la secuencia de inserción.

3. ¿Qué ha pasado en el TreeMap?
Los nombres están ordenados de la A a la Z. Al basarse en un árbol binario, ordena automáticamente las llaves por su orden natural.

4. Caso práctico: Agenda telefónica
Elegiría el TreeMap. Es la opción ideal para una agenda ya que permite realizar búsquedas rápidas y garantiza que los contactos estén siempre ordenados alfabéticamente sin intervención manual.
