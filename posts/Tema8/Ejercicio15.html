Ejercicio 15: Collections - Utilidades Imprescindibles

Enunciado

En este ejercicio final del tema, aprenderemos a utilizar la clase Collections. Es importante no confundirla con la interfaz Collection; Collections (con 's') es una clase de utilidad que contiene métodos estáticos para operar sobre nuestras colecciones.

Objetivos:

  • Gestionar una lista de puntuaciones numéricas[cite: 1].
  • Ordenar: Clasificar los datos de menor a mayor de forma automática.
  • Estadísticas: Obtener los valores extremos sin necesidad de bucles.
  • Aleatoriedad: Barajar la lista para simular un sorteo.

Solución

Utilizaremos una lista de enteros para poner a prueba los métodos más comunes de esta clase de utilidad. Observa cómo cambia la lista después de cada llamada a Collections.

package Tema8.Ejercicios.ejercicio15;

import java.util.ArrayList;
import java.util.Collections;
import java.util.List;

public class Ejercicio15 {
    public static void main(String[] args) {
        // 1. Crear lista con 6 puntuaciones desordenadas
        List<Integer> puntuaciones = new ArrayList<>();
        puntuaciones.add(450);
        puntuaciones.add(1200);
        puntuaciones.add(300);
        puntuaciones.add(800);
        puntuaciones.add(1500);
        puntuaciones.add(100);

        System.out.println("Lista inicial: " + puntuaciones);

        // 2. Ordenar de menor a mayor usando Collections.sort()
        Collections.sort(puntuaciones);
        System.out.println("Lista ordenada: " + puntuaciones);

        // 3. Buscar la puntuación más alta y la más baja (max y min)
        int max = Collections.max(puntuaciones);
        int min = Collections.min(puntuaciones);

        System.out.println("Puntuación más alta: " + max);
        System.out.println("Puntuación más baja: " + min);

        // 4. Desordenar la lista (sorteo) usando Collections.shuffle()
        Collections.shuffle(puntuaciones);
        System.out.println("Lista tras el sorteo (shuffle): " + puntuaciones);
    }
}

Diferencia Clave

Collection vs Collections:
Es un error común en los exámenes: Collection es la interfaz que define qué es una colección. En cambio, Collections es una clase de utilidades que proporciona "herramientas" (métodos estáticos) para trabajar con ellas, como ordenar o buscar.
