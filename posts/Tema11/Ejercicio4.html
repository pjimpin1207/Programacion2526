Ejercicio 4: El Panel de un Ascensor — Proporcionalidad con GridLayout

Enunciado y Desafío

A diferencia del posicionamiento absoluto por píxeles fijos estudiado en prácticas anteriores, la creación de interfaces de usuario profesionales exige que el software se adapte de forma dinámica a las acciones del usuario. En este ejercicio diseñaremos el panel de mandos digital para un ascensor de cuatro plantas.

Restricciones de maquetación:

  • Los botones de llamada a los pisos deben organizarse en una cuadrícula geométrica perfecta compuesta exactamente por 2 filas y 2 columnas.
  • Comportamiento elástico: Si el usuario estira, maximiza o deforma las dimensiones de la ventana principal, los botones de control internos deben expandirse y redimensionarse de forma automática manteniendo una proporcionalidad exacta.
---

Solución Completa

Para cumplir con el requerimiento de proporcionalidad y división matricial, implementamos la clase GridLayout. Este gestor anula las dimensiones preferidas individuales e impone una división matemática equitativa sobre toda la superficie útil del contenedor.

package Tema11.Ejercicios.ejercicio4;

import java.awt.*;

public class Ejercicio4 {

    public static void main(String[] args) {
        // 1. Inicializar el contenedor de nivel superior asignándole dimensiones iniciales
        Frame frame = new Frame("Panel de Ascensor");
        frame.setSize(300, 400);

        // 2. CONFIGURACIÓN DEL ADMINISTRADOR DE DISEÑO (Layout Manager)
        // Instanciamos un GridLayout parametrizado formalmente con (filas, columnas)
        frame.setLayout(new GridLayout(2, 2));

        // 3. Crear e inyectar las instancias de los botones directamente al Frame
        // El orden de inserción rellena la matriz de izquierda a derecha y de arriba a abajo
        frame.add(new Button("Piso 3")); // Fila 1 - Columna 1
        frame.add(new Button("Piso 4")); // Fila 1 - Columna 2
        frame.add(new Button("Piso 1")); // Fila 2 - Columna 1
        frame.add(new Button("Piso 2")); // Fila 2 - Columna 2

        // 4. Renderizar y mostrar la interfaz gráfica finalizada en pantalla
        frame.setVisible(true);
    }
}

Análisis del Comportamiento Elástico Matricial

¿Por qué es la solución perfecta para este escenario?

  • Garantía de proporcionalidad: Cuando el usuario estira el borde del Frame, el motor gráfico de AWT recalcula el espacio interior del lienzo (ancho y alto utilizables) y lo divide de forma estricta entre el número de filas y columnas declaradas (2x2). Como resultado, cada botón recibe exactamente el **25% de la superficie disponible**, escalando su tamaño de forma 100% responsiva.
  • Flujo indexado de celdas: El método add() genérico no requiere que le indiquemos coordenadas cartesianas (como norte o sur). Las etiquetas se posicionan siguiendo la dirección de lectura de una cuadrícula: se completa la primera línea horizontal y, automáticamente, el siguiente componente salta al inicio de la segunda fila.
  • Componentes pesados y estiramiento: Al usar componentes nativos de AWT (Heavyweight components), la fuente tipográfica interna del texto del botón mantendrá su tamaño fijo, pero el cuerpo contenedor que rodea al botón reaccionará adaptándose de forma flexible a la geometría del monitor del usuario.
