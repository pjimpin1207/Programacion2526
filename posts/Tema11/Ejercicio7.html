Ejercicio 7: Formulario de Login Avanzado con Diálogos Modales y Selector de Archivos

Enunciado y Requisitos

En esta práctica consolidamos todo lo aprendido sobre maquetación absoluta y control distributed de eventos. Construiremos un panel de autenticación de usuario (Login) completamente funcional que cumpla con los estándares operativos de seguridad e interacción secundaria.

Especificaciones técnicas obligatorias:

  • Entrada segura: Campos para usuario y clave (ocultando los caracteres de la contraseña mediante una máscara visual).
  • Control de Ventana: Habilitar el cierre nativo del contenedor principal.
  • Limpieza en cascada: Restablecer de manera automática el texto de todo el formulario pulsando un botón dedicado.
  • Flujo modal: Validar los datos de entrada al pulsar "Aceptar" y levantar un cuadro de diálogo secundario de tipo modal que dé la bienvenida personalizada al usuario.
  • Misión Opcional (Ampliación): Incorporar un disparador que invoque al explorador de archivos nativo del sistema, capturando y mostrando la ruta absoluta del fichero seleccionado.
---

Solución Completa

Combinamos el uso de setEchoChar('*') para la protección de la clave, la clase Dialog parametrizada en true para el bloqueo modal de la interfaz, y la clase utilitaria FileDialog para interactuar de forma directa con los ficheros del sistema operativo.

package Tema11.Ejercicios.ejercicio7;

import java.awt.*;
import java.awt.event.*;

public class Ejercicio7 extends Frame implements ActionListener {

    // Declaración de componentes globales de la vista
    private TextField txtUsuario;
    private TextField txtPass;
    private Button btnAceptar;
    private Button btnLimpiar;
    private Button btnFichero;
    private Label lblRuta;

    public Ejercicio7() {
        super("Pantalla de Login");

        // Desactivamos el gestor automático para posicionar de forma milimétrica por píxeles
        setLayout(null);
        setSize(400, 320);

        // 1. Zona de Usuario
        Label lblUsuario = new Label("Usuario:");
        lblUsuario.setBounds(50, 50, 80, 30);
        txtUsuario = new TextField();
        txtUsuario.setBounds(140, 50, 180, 30);

        // 2. Zona de Contraseña (Segura)
        Label lblPass = new Label("Contraseña:");
        lblPass.setBounds(50, 100, 80, 30);
        txtPass = new TextField();
        txtPass.setEchoChar('*'); // Enmascara el texto introducido sustituyendo las letras por asteriscos
        txtPass.setBounds(140, 100, 180, 30);

        // 3. Botonera de Control
        btnAceptar = new Button("Aceptar");
        btnAceptar.setBounds(50, 160, 100, 30);

        btnLimpiar = new Button("Limpiar");
        btnLimpiar.setBounds(160, 160, 100, 30);

        // 4. Zona de Ampliación (Explorador de Archivos)
        btnFichero = new Button("Seleccionar Fichero");
        btnFichero.setBounds(50, 210, 150, 30);

        lblRuta = new Label("Ningún fichero seleccionado");
        lblRuta.setBounds(50, 250, 320, 30);

        // REGISTRO DE ESCUCHADORES: Enlazamos los tres disparadores al gestor unificado 'this'
        btnAceptar.addActionListener(this);
        btnLimpiar.addActionListener(this);
        btnFichero.addActionListener(this);

        // Controlador defensivo para la destrucción del proceso al cerrar la ventana
        addWindowListener(new WindowAdapter() {
            @Override
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });

        // Acoplamiento secuencial de elementos al lienzo principal
        add(lblUsuario);
        add(txtUsuario);
        add(lblPass);
        add(txtPass);
        add(btnAceptar);
        add(btnLimpiar);
        add(btnFichero);
        add(lblRuta);

        // Centrar ventana en la pantalla y renderizar
        setLocationRelativeTo(null);
        setVisible(true);
    }

    // INTERCEPTOR DE ACCIONES: Enrutamiento lógico según el emisor del evento
    @Override
    public void actionPerformed(ActionEvent e) {
        
        // CASO A: Pulsación en "Aceptar" -> Inicializar el Diálogo de Bienvenida
        if (e.getSource() == btnAceptar) {
            String usuarioStr = txtUsuario.getText();

            // CONTRATO MODAL: Instanciamos el diálogo pasando (VentanaPadre, Título, Modal=true)
            // Al ser modal (true), congela y bloquea la ventana de Login de fondo hasta que se destruya
            Dialog dialogoBienvenida = new Dialog(this, "Mensaje", true);
            dialogoBienvenida.setLayout(new FlowLayout());
            dialogoBienvenida.setSize(250, 100);

            // Componentes internos del contenedor flotante
            Label lblMensaje = new Label("¡Bienvenido, " + usuarioStr + "!");
            Button btnCerrarDialogo = new Button("OK");

            // Evento embebido: El botón OK libera y destruye la tarjeta de memoria del diálogo
            btnCerrarDialogo.addActionListener(new ActionListener() {
                public void actionPerformed(ActionEvent evt) {
                    dialogoBienvenida.dispose(); // Cierra el diálogo y desbloquea el login
                }
            });

            dialogoBienvenida.add(lblMensaje);
            dialogoBienvenida.add(btnCerrarDialogo);
            dialogoBienvenida.setLocationRelativeTo(this); // Sincroniza la posición sobre el padre
            dialogoBienvenida.setVisible(true);
        }

        // CASO B: Pulsación en "Limpiar" -> Vaciado inmediato de buffers
        else if (e.getSource() == btnLimpiar) {
            txtUsuario.setText("");
            txtPass.setText("");
        }

        // CASO C: Pulsación en "Seleccionar Fichero" -> Invocar ventana nativa del S.O.
        else if (e.getSource() == btnFichero) {
            FileDialog fd = new FileDialog(this, "Elige un archivo", FileDialog.LOAD);
            fd.setVisible(true); // Se bloquea aquí el hilo hasta que el usuario elija o cancele

            // Validamos que el usuario no haya cerrado el explorador sin escoger nada
            if (fd.getFile() != null) {
                String rutaCompleta = fd.getDirectory() + fd.getFile();
                lblRuta.setText("Archivo: " + rutaCompleta);
            }
        }
    }

    public static void main(String[] args) {
        new Ejercicio7();
    }
}

Análisis de Componentes Avanzados de Interacción

Claves de diseño e ingeniería de interfaces aplicadas en esta solución:

  • Seguridad en formularios (setEchoChar): Por diseño, los campos TextField muestran los caracteres en texto plano. Invocar a setEchoChar('*') reconfigura el comportamiento del búfer visual del sistema de ventanas, ocultando los caracteres de entrada frente a miradas indiscretas, manteniendo el texto íntegro al leerlo con getText().
  • El concepto de Ventana Modal (Dialog): Al pasar el parámetro booleano en true dentro del constructor new Dialog(this, "Título", true), creamos una ventana modal de bloqueo. Esto congela de forma inmediata la cola de eventos de la ventana de login subyacente. El usuario no podrá hacer clic, rellenar campos o mover la pantalla principal hasta que atienda y descarte el diálogo (ejecutando dispose()), asegurando el flujo secuencial de la aplicación.
  • Interoperabilidad con FileDialog: A diferencia de maquetar un explorador manualmente, la clase FileDialog de AWT es un componente de "peso pesado" (Heavyweight component). Delega la llamada directamente en el núcleo del Sistema Operativo host. Mostrará la ventana de navegación nativa de Windows, macOS o Linux según donde se ejecute, capturando de forma segura las cadenas de texto del directorio (getDirectory()) y del archivo (getFile()).
