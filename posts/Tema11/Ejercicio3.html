Ejercicio 3: Dominando la Distribución con Layout Managers

Enunciado y Contexto

En el desarrollo de software de escritorio, delegamos la colocación y el dimensionamiento de los componentes lógicos en los Layout Managers. Estos administradores de diseño calculan de forma elástica la geometría de las interfaces, garantizando que el diseño sea multi-plataforma y responsivo ante cambios de tamaño en las ventanas.

Misión planteada:

Implementar un ejemplo práctico e independiente para cada uno de los cuatro administradores de distribución esenciales del paquete estándar java.awt:

  • BorderLayout: Distribución en regiones cardinales y una zona central extensiva.
  • CardLayout: Estructura en pila de tarjetas, mostrando un único lienzo visible a la vez.
  • FlowLayout: Flujo de componentes en serie con saltos automáticos de línea.
  • GridLayout: Disposición geométrica simétrica en forma de cuadrícula matricial de filas y columnas.

Implementación de los Diseños

1. BorderLayout

Organiza los componentes en cinco regiones delimitadas: Norte, Sur, Este, Oeste y Centro. El área central absorbe de forma automática todo el espacio residual no ocupado por los bloques cardinales.

package Tema11.Ejercicios.ejercicio3;

import java.awt.*;

public class EjemploBorderLayout {
    public static void main(String[] args) {
        Frame frame = new Frame("Ejemplo BorderLayout");
        frame.setSize(400, 300);

        // Establecer de forma explícita el gestor BorderLayout
        frame.setLayout(new BorderLayout());

        // Añadir componentes indicando la región cardinal exacta de destino
        frame.add(new Button("Norte (NORTH)"), BorderLayout.NORTH);
        frame.add(new Button("Sur (SOUTH)"), BorderLayout.SOUTH);
        frame.add(new Button("Este (EAST)"), BorderLayout.EAST);
        frame.add(new Button("Oeste (WEST)"), BorderLayout.WEST);
        frame.add(new Button("Centro (CENTER) - Ocupa el resto"), BorderLayout.CENTER);

        frame.setVisible(true);
    }
}

2. CardLayout

Trata el contenedor como una pila de cartas ocultas donde solo la de la parte superior permanece visible. En este ejemplo, acoplamos un escuchador de eventos interactivo (ActionListener) para alternar dinámicamente entre las tarjetas lógicas mediante el método next().

package Tema11.Ejercicios.ejercicio3;

import java.awt.*;
import java.awt.event.*;

public class EjemploCardLayout {
    public static void main(String[] args) {
        Frame frame = new Frame("Ejemplo CardLayout");
        frame.setSize(300, 200);

        // 1. Instanciar el gestor y el panel contenedor secundario que albergará las cartas
        CardLayout cardLayout = new CardLayout();
        Panel panelCartas = new Panel(cardLayout);

        // 2. Insertar los componentes asignándoles un alias o etiqueta de texto única
        panelCartas.add(new Button("Esta es la Carta 1"), "CARTA_1");
        panelCartas.add(new Button("Esta es la Carta 2"), "CARTA_2");
        panelCartas.add(new Button("Esta es la Carta 3"), "CARTA_3");

        // Ubicamos el mazo de cartas en el centro de la ventana principal
        frame.add(panelCartas, BorderLayout.CENTER);

        // 3. Crear el botón de control en la base de la ventana
        Button btnSiguiente = new Button("Siguiente Tarjeta >>");
        frame.add(btnSiguiente, BorderLayout.SOUTH);

        // 4. Acoplar un controlador de eventos para rotar las tarjetas al pulsar el botón
        btnSiguiente.addActionListener(new ActionListener() {
            public void actionPerformed(ActionEvent e) {
                cardLayout.next(panelCartas); // Salta secuencialmente a la siguiente carta
            }
        });

        frame.setVisible(true);
    }
}

3. FlowLayout

Alinea los componentes horizontalmente en una fila respetando sus dimensiones preferidas de fábrica. Si la anchura física del Frame no es suficiente para albergar un nuevo elemento, el gestor realiza un salto de línea automático.

package Tema11.Ejercicios.ejercicio3;

import java.awt.*;

public class EjemploFlowLayout {
    public static void main(String[] args) {
        Frame frame = new Frame("Ejemplo FlowLayout");
        frame.setSize(400, 150);

        // Establecemos el FlowLayout (por defecto alinea los elementos al centro)
        frame.setLayout(new FlowLayout());

        // Añadimos varios botones para auditar cómo fluyen de forma adaptativa
        frame.add(new Button("Botón 1"));
        frame.add(new Button("Botón 2"));
        frame.add(new Button("Un botón con texto mas largo"));
        frame.add(new Button("Botón 4"));
        frame.add(new Button("Botón 5"));

        frame.setVisible(true);
    }
}

4. GridLayout

Divide la superficie útil del contenedor en una cuadrícula rígida de celdas idénticas organizadas en filas y columnas. Cada componente insertado se estira de forma forzada para ocupar la totalidad de la celda asignada.

package Tema11.Ejercicios.ejercicio3;

import java.awt.*;

public class EjemploGridLayout {
    public static void main(String[] args) {
        Frame frame = new Frame("Ejemplo GridLayout");
        frame.setSize(300, 200);

        // Inicializamos una cuadrícula restrictiva de 2 filas y 3 columnas
        frame.setLayout(new GridLayout(2, 3));

        // Las inserciones rellenan de izquierda a derecha la fila actual y luego saltan a la siguiente
        frame.add(new Button("F1 - C1"));
        frame.add(new Button("F1 - C2"));
        frame.add(new Button("F1 - C3"));
        frame.add(new Button("F2 - C1"));
        frame.add(new Button("F2 - C2"));
        frame.add(new Button("F2 - C3"));

        frame.setVisible(true);
    }
}

Comparativa de Comportamientos Estructurales

  • BorderLayout es el esqueleto idóneo para la estructura principal de la aplicación. Nos permite anclar barras de herramientas fijas arriba (NORTH), estados de estado abajo (SOUTH), paneles de navegación laterales y dejar un Lienzo central dinámico (CENTER).
  • CardLayout es el núcleo tecnológico detrás de los asistentes de configuración de pasos múltiples (Wizards "Siguiente/Atrás") o de las interfaces compuestas por pestañas de opciones.
  • FlowLayout respeta fielmente el tamaño mínimo del contenido de los botones, haciéndolo ideal para agrupar conjuntos de botones pequeños dentro de paneles subordinados.
  • GridLayout ignora las dimensiones preferidas de los componentes e impone una simetría matemática absoluta sobre el contenedor. Es la estructura estándar para diseñar teclados numéricos de calculadoras, tableros de juegos o matrices de botones uniformes.
