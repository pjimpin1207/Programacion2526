Ejercicio 5: Creación de Menús de Edición con la API AWT

Enunciado y Objetivos

Las aplicaciones profesionales de escritorio organizan sus acciones complejas a través de un sistema de navegación superior. El objetivo de este ejercicio es diseñar la estructura visual de menús desplegables para simular las herramientas de formato básicas de un procesador de textos.

Requisitos del sistema:

  • Crear una ventana provista de una etiqueta central (Label) con alineación justificada que contenga el mensaje "Texto de prueba".
  • Diseñar e inyectar una barra de menú superior (MenuBar).
  • Implementar la pestaña desplegable "Estilo" con las opciones interactivas "Negrita" y "Normal".
  • Implementar la pestaña desplegable "Color" provista de los botones de selección "Rojo", "Azul" y "Negro".
---

Solución Completa

Para ensamblar menús en Java AWT, debemos seguir una jerarquía estricta de herencia: las opciones individuales (MenuItem) se añaden a los contenedores desplegables (Menu), los cuales a su vez se acoplan a la barra contenedora principal (MenuBar) vinculada al marco mediante setMenuBar().

package Tema11.Ejercicios.ejercicio5;

import java.awt.*;
import java.awt.event.*;

public class Ejercicio5 {
    public static void main(String[] args) {

        // 1. Inicializar la ventana contenedora principal
        Frame frame = new Frame("Editor de Texto");
        frame.setSize(400, 300);
        frame.setLayout(new BorderLayout()); // Usamos BorderLayout para centrar la etiqueta

        // 2. Creación y acoplamiento de la etiqueta de texto central
        Label lblTexto = new Label("Texto de prueba", Label.CENTER);
        frame.add(lblTexto, BorderLayout.CENTER);

        // 3. ESTRUCTURA DE MENÚS: Instanciar la barra de herramientas superior
        MenuBar barraMenu = new MenuBar();

        // 4. Configuración de la pestaña desplegable "Estilo"
        Menu menuEstilo = new Menu("Estilo");
        MenuItem itemNegrita = new MenuItem("Negrita");
        MenuItem itemNormal = new MenuItem("Normal");
        
        // Colgamos las opciones hijas del menú padre Estilo
        menuEstilo.add(itemNegrita);
        menuEstilo.add(itemNormal);

        // 5. Configuración de la pestaña desplegable "Color"
        Menu menuColor = new Menu("Color");
        MenuItem itemRojo = new MenuItem("Rojo");
        MenuItem itemAzul = new MenuItem("Azul");
        MenuItem itemNegro = new MenuItem("Negro");
        
        // Colgamos las opciones hijas del menú padre Color
        menuColor.add(itemRojo);
        menuColor.add(itemAzul);
        menuColor.add(itemNegro);

        // 6. Enlazar los menús desplegables a la barra principal superior
        barraMenu.add(menuEstilo);
        barraMenu.add(menuColor);

        // 7. Acoplar de forma física la barra de menús estructurada al marco de la ventana
        frame.setMenuBar(barraMenu);
        
        // Renderizar la interfaz gráfica
        frame.setVisible(true);
    }
}

Análisis Jerárquico de Componentes de Menú

Comprendiendo la arquitectura de contenedores superiores en AWT:

Un error conceptual común al empezar a maquetar menús es tratar la barra de herramientas como si fuera un componente común (como un botón o una etiqueta) e intentar inyectarla mediante frame.add(). En Java, los menús siguen una línea de herencia especial al margen de la clase Component:

  • MenuBar: Representa el lienzo o la franja horizontal física que se posiciona de forma fija inmediatamente debajo de la barra de título de la ventana. Solo puede existir una instancia de MenuBar asociada a cada Frame, y su acoplamiento se realiza mediante el método específico frame.setMenuBar(barraMenu).
  • Menu: Representa los bloques o pestañas de texto que se sitúan sobre la barra (como "Estilo" o "Color"). Su comportamiento nativo consiste en desplegar una lista vertical flotante al hacer clic sobre ellos.
  • MenuItem: Representa los elementos finales u hojas de la jerarquía. Son las opciones clicables sobre las que el usuario interactúa para desencadenar acciones en el programa (como "Negrita" o "Rojo").
