Ejercicio 10: Acceso Secuencial y Adición de Contenido

Enunciado

En este ejercicio consolidamos los flujos de acceso secuencial trabajando con archivos de texto plano. El objetivo es interactuar de forma segura con un fichero físico para añadir información sin destruir la ya existente y leer el resultado completo carácter por carácter.

Requisitos obligatorios:

  • Integrar la clase File para verificar la existencia del archivo (ejercicio10.txt) y validar que es un fichero correcto antes de abrir canales de comunicación.
  • Utilizar un flujo de salida que permita añadir contenido al final del archivo de texto plano de manera no destructiva (modo append).
  • Leer secuencialmente el contenido del archivo carácter por carácter controlando la marca del final del flujo (-1) y mostrarlo por la consola.
---

Solución Completa

Empleamos FileWriter pasando el parámetro booleano true para activar el modo de adición, y FileReader para el bucle secuencial de entrada de caracteres, todo gestionado con estructuras try-with-resources.

package Tema9.Ejercicios.ejercicio10;

import java.io.*;

public class Ejercicio10 {
    public static void main(String[] args) {
        // 1. Instanciar la ruta abstracta del archivo
        File archivo = new File("ejercicio10.txt");

        // 2. Verificar la existencia y la validez estructural del recurso
        if (archivo.exists() && archivo.isFile()) {
            System.out.println("Archivo encontrado: " + archivo.getName());
            System.out.println("Tamaño inicial: " + archivo.length() + " bytes");

            // 3. Acceso secuencial de escritura (Modo Append)
            // Pasamos 'true' como segundo argumento para añadir al final sin sobreescribir
            try (FileWriter escritor = new FileWriter(archivo, true)) {
                escritor.write("\nEsta línea usando acceso secuencial");
                System.out.println("Contenido añadido correctamente.");
            } catch (IOException e) {
                System.err.println("Error al escribir: " + e.getMessage());
            }

            // 4. Acceso secuencial de lectura carácter por carácter
            System.out.println("\n--- Contenido actual del fichero ---");
            try (FileReader lector = new FileReader(archivo)) {
                int caracter;
                // El método read() consume un carácter y devuelve su valor entero ASCII
                // Devuelve -1 de forma estricta al alcanzar el final del flujo (EOF)
                while ((caracter = lector.read()) != -1) {
                    System.out.print((char) caracter); // Casting explícito a tipo char
                }
                System.out.println("\n\nLectura finalizada. Tamaño actual: " + archivo.length() + " bytes");
            } catch (IOException e) {
                System.err.println("Error al leer: " + e.getMessage());
            }

        } else {
            System.out.println("El archivo 'ejercicio10.txt' no existe o la ruta no es válida.");
        }
    }
}

Análisis de Conceptos Clave

Mecánica del Acceso Secuencial:

  • El constructor en modo Append: Por defecto, si instanciamos un new FileWriter("ruta.txt"), Java asume que queremos crear un archivo nuevo o truncar el existente (borrando todo su contenido). Al añadir el parámetro opcional true (new FileWriter(file, true)), el puntero de escritura del sistema operativo se posiciona automáticamente al final del último byte registrado, permitiendo la concatenación.
  • El Centinela -1: El método read() de la clase FileReader devuelve un tipo int en lugar de un char directamente. Esto se debe a que necesita un rango de valores extra para representar una condición especial: el **End of File (EOF)**, el cual está codificado universalmente en Java con el valor numérico -1.
  • Garantía de flujos con try-with-resources: Al declarar los flujos de lectura y escritura entre los paréntesis de la sentencia try, nos aseguramos por contrato de que Java liberará de manera automática los canales de comunicación físicos invocando al método close(), ocurra o no un fallo en tiempo de ejecución.
