Ejercicio 8: Lectura Avanzada con BufferedReader (Mark y Reset)

Enunciado

A partir de un fichero de texto plano (como por ejemplo nota.txt), el objetivo de esta práctica es implementar un lector de alto nivel capaz de manipular la posición del puntero de lectura mediante buffers, permitiendo releer líneas de datos sin necesidad de cerrar y volver a abrir el flujo.

---

Solución Completa

Hacemos uso del bloque moderno try-with-resources para asegurar la apertura e inserción segura de los flujos de datos de caracteres, controlando dinámicamente si el flujo soporta el mecanismo de marcas del buffer.

package Tema9.Ejercicios.ejercicio8;

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class Ejercicio8 {
    public static void main(String[] args) {
        // Uso de try-with-resources para garantizar el cierre automático de los descriptores de archivo
        try (BufferedReader br = new BufferedReader(new FileReader("nota.txt"))) {
            
            // 1. Validamos de forma condicional si la implementación del flujo admite marcas
            if (br.markSupported()) {
                // 2. Fijamos un punto de restauración en la posición actual del buffer (límite de 100 caracteres)
                br.mark(100); 
                
                System.out.println("Primera línea: " + br.readLine());
                
                // 3. Rebobinamos el flujo volviendo al punto de control previamente guardado
                br.reset(); 
                
                // 4. Repetimos la lectura para comprobar que se extraen de nuevo los mismos datos
                System.out.println("Repetimos línea: " + br.readLine());
            }
            
        } catch (IOException e) { 
            e.printStackTrace(); 
        }
    }
}

Análisis de la Mecánica de Buffers

El poder del almacenamiento intermedio:

Por norma general, un flujo de datos (stream o reader) es secuencial y unidireccional: un carácter consumido se pierde del canal de entrada y el lector avanza hacia adelante de manera indefinida.

  • BufferedReader: Envuelve un flujo plano como FileReader y precarga bloques de texto en la memoria RAM del sistema. Gracias a esto, expone métodos utilitarios de alto rendimiento como readLine().
  • mark(int readAheadLimit): Le indica al búfer que retenga internamente en memoria los caracteres que se lean a partir de este instante. El parámetro numérico determina cuántos caracteres máximos podemos leer antes de que la marca quede invalidada por el recolector de basura.
  • reset(): Regresa el puntero lógico de lectura a la posición exacta donde se invocó mark(), permitiendo "repetir" la lectura de datos de manera idéntica y transparente.
