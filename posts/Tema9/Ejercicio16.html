Ejercicio 16: Escritura de Archivos XML con la API DOM

Enunciado

En este ejercicio exploraremos el proceso inverso al parseo: la **creación desde cero de un documento estructurado XML**. El objetivo es diseñar un generador automático para el fichero de configuración de un videojuego denominado juego_config.xml.

Estructura del archivo objetivo:

La aplicación debe generar de manera física un archivo en la raíz del proyecto que replique fielmente el siguiente árbol de marcas indentado:

<videojuego>
    <pantalla resolucion="1920x1080">true</pantalla>
    <audio volumen="80">false</audio>
</videojuego>
---

Solución Completa

El desarrollo se divide en tres fases lógicas: inicializar un lienzo en blanco en la RAM, ensamblar la jerarquía de nodos colgándolos de sus respectivos padres y, finalmente, invocar a la clase de transformación Transformer para volcar el árbol lógico al disco duro.

package Tema9.Ejercicios.ejercicio16;

import org.w3c.dom.Document;
import org.w3c.dom.Element;

import javax.xml.parsers.DocumentBuilderFactory;
import javax.xml.transform.OutputKeys;
import javax.xml.transform.Transformer;
import javax.xml.transform.TransformerFactory;
import javax.xml.transform.dom.DOMSource;
import javax.xml.transform.stream.StreamResult;
import java.io.File;

public class Ejercicio16 {
    public static void main(String[] args) {
        try {
            // === FASE 1: Inicializar el documento vacío en memoria RAM ===
            DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
            DocumentBuilder builder = factory.newDocumentBuilder();
            // Creamos una instancia de objeto Document en blanco
            Document documento = builder.newDocument();

            // === FASE 2: Construir el árbol de Nodos y Elementos ===

            // 1. Creación del elemento raíz 
            Element raiz = documento.createElement("videojuego");
            // Acoplamos la raíz directamente al nodo principal del documento
            documento.appendChild(raiz);

            // 2. Creación del elemento secundario 
            Element pantalla = documento.createElement("pantalla");
            // Asignación de su atributo estructurado llave-valor
            pantalla.setAttribute("resolucion", "1920x1080");
            // Inserción de la información de texto interna del nodo
            pantalla.setTextContent("true");
            
            // ¡Crucial! Colgamos el nodo pantalla del elemento raíz para que no flote en memoria
            raiz.appendChild(pantalla);

Análisis y Conceptos Fundamentales de la Creación DOM

La importancia del enlazado de ramas (appendChild):

  • documento.createElement(String name): Este método actúa como una fábrica de etiquetas virtuales. Instancia el elemento en la memoria volátil, pero de manera aislada. Si nos limitamos a crear elementos sin invocar a appendChild(), los nodos se quedarán "flotando" en la memoria RAM desvinculados del documento final.
  • Estructura en árbol: El orden de enlazado determina la jerarquía. La etiqueta raíz se cuelga directamente del objeto base (documento.appendChild(raiz)), mientras que las etiquetas de configuración específicas se cuelgan de la raíz (raiz.appendChild(pantalla)).
  • La API Transformer: Por diseño, el DOM solo sabe gestionar estructuras abstractas en memoria. Para traducir ese árbol de objetos a una secuencia de texto legible que un disco duro entienda, recurrimos a un motor de transformación (Transformer).
  • El truco de la indentación: Por defecto, un exportador transforma los nodos en una única línea masiva de texto comprimido para ahorrar espacio. Al activar la propiedad de salida OutputKeys.INDENT junto con la directiva específica de sangrado a 4 espacios de Apache, forzamos al sistema a formatear el archivo con saltos de línea estéticos, haciéndolo legible para los seres humanos.
