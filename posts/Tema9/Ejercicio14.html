Ejercicio 14: Serialización y Recuperación de Colecciones Dinámicas

Enunciado

En este ejercicio avanzamos en el uso de flujos de objetos. En lugar de persistir una única instancia independiente, aprenderemos a congelar y restaurar una estructura de datos completa (un ArrayList) que almacena múltiples objetos de nuestro propio tipo personalizado.

Fases del reto:

  • Crear: Diseñar una clase modelo llamada Item con las propiedades nombre, peso y valor que implemente el contrato Serializable.
  • Guardar: Generar una lista con 5 elementos guardándola íntegramente en un archivo binario indexado denominado mochila.bin.
  • Recuperar: Crear un programa independiente que lea el archivo, reconstruyendo la estructura y calculando iterativamente el valor total acumulado de los elementos.
---

Estructura de Clases de la Solución

1. La Clase Molde (Item)

Definimos los atributos encapsulados y redefinimos el método estándar toString() para que, al volcar la lista por consola durante la recuperación, los datos se presenten de manera amigable.

package Tema9.Ejercicios.ejercicio14;

import java.io.Serializable;

public class Item implements Serializable {
    private static final long serialVersionUID = 1L;

    private String nombre;
    private Double peso;
    private int valor;

    public Item(String nombre, double peso, int valor) {
        this.nombre = nombre;
        this.peso = peso;
        this.valor = valor;
    }

    public String getNombre() { return nombre; }
    public void setNombre(String nombre) { this.nombre = nombre; }

    public double getPeso() { return peso; }
    public void setPeso(double peso) { this.peso = peso; }

    public int getValor() { return valor; }
    public void setValor(int valor) { this.valor = valor; }

    @Override
    public String toString() {
        return "Item [Nombre: " + nombre + ", Peso: " + peso + "kg, Valor: " + valor + " monedas]";
    }
}

2. Programa de Grabación (Ejercicio14)

Instanciamos nuestra colección dinámica y pasamos el objeto List completo al método writeObject(). Al ser las colecciones nativas de Java (como ArrayList) clases serializables por defecto, guardarán automáticamente todos los elementos que contengan.

package Tema9.Ejercicios.ejercicio14;

import java.io.FileOutputStream;
import java.io.IOException;
import java.io.ObjectOutputStream;
import java.util.ArrayList;
import java.util.List;

public class Ejercicio14 {
    public static void main(String[] args) {
        // Creamos la mochila y añadimos 5 ítems de inventario
        List<Item> mochila = new ArrayList<>();
        mochila.add(new Item("Manzanas", 2.3, 50));
        mochila.add(new Item("Pizza", 3.5, 25));
        mochila.add(new Item("Patatas", 7.5, 7));
        mochila.add(new Item("Macarrones", 1.2, 3));
        mochila.add(new Item("Queso", 0.4, 10));

        try {
            // Inicialización de flujos manteniendo las declaraciones en líneas separadas
            FileOutputStream fichero = new FileOutputStream("mochila.bin");
            ObjectOutputStream out = new ObjectOutputStream(fichero);
            
            // Serializamos la lista completa con un único método
            out.writeObject(mochila);
            
            out.close();
            System.out.println("Colección de la mochila serializada en 'mochila.bin'");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

3. Programa de Extracción (RecuperarMochila)

Leemos el flujo binario aplicando el correspondiente moldeo o casting estructural a la lista genérica y sumamos las propiedades numéricas mediante un bucle for-each.

package Tema9.Ejercicios.ejercicio14;

import java.io.FileInputStream;
import java.io.IOException;
import java.io.ObjectInputStream;
import java.util.List;

public class RecuperarMochila {
    public static void main(String[] args) {
        try {
            // Vinculamos y abrimos el canal de lectura binaria
            FileInputStream fichero = new FileInputStream("mochila.bin");
            ObjectInputStream in = new ObjectInputStream(fichero);
            
            // Reconstruimos el objeto completo recuperando la estructura de la lista
            @SuppressWarnings("unchecked")
            List<Item> mochilaRecuperada = (List<Item>) in.readObject();
            in.close();

            // Recorremos la lista recuperada para auditar su valor económico total
            int valorTotal = 0;
            for (Item i : mochilaRecuperada) {
                valorTotal += i.getValor();
            }
            
            System.out.println("--- Contenido de la Mochila Recuperada ---");
            System.out.println(mochilaRecuperada);
            System.out.println("\nValor total de la mochila: " + valorTotal + " monedas.");

        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}
---

Pregunta de Reflexión Teórica

¿Qué pasa si intentas guardar un objeto de una clase que NO es Serializable?

Si la clase Item no tuviera el marcador implícito implements Serializable, el compilador de Java no mostraría ningún error previo. Sin embargo, en tiempo de ejecución, en el instante exacto en que el flujo invoque a out.writeObject(mochila), el programa fallará de forma abrupta lanzando una excepción de tipo java.io.NotSerializableException.

Esto ocurre porque Java protege la seguridad y consistencia del entorno, denegando de forma estricta el volcado a disco de cualquier objeto que no haya dado su autorización explícita a ser transformado en flujos binarios.
