Ejercicio 5: Hola Mundo interactivo con control de errores

Enunciado

En este ejercicio volvemos a la clase de alto nivel Scanner para crear un flujo interactivo. Debemos solicitar al usuario su nombre completo y su edad, asegurando un control estricto de errores para evitar que la aplicación falle si se introduce un tipo de dato incorrecto.

Requisitos:

  • Capturar el nombre completo (incluyendo espacios).
  • Solicitar la edad controlando mediante excepciones que sea obligatoriamente un número entero.
  • Mantener al usuario en un bucle de repetición hasta que la entrada de la edad sea válida.

Solución Completa

Para recuperar cadenas de texto con espacios usamos nextLine(), y para asegurar la integridad de los datos numéricos empleamos un bloque try-catch capturando la excepción InputMismatchException.

package Tema9.Ejercicios.ejercicio5;

import java.util.Scanner;
import java.util.InputMismatchException;

public class Ejercicio5 {
    public static void main(String[] args) {
        // Crea el objeto Scanner asociado a la entrada estándar de consola
        Scanner sc = new Scanner(System.in);

        // 1. Pedir nombre completo
        System.out.println("Escribe tu nombre completo:");
        // Uso nextLine() para capturar toda la frase hasta pulsar Enter (incluyendo espacios)
        String nombre = sc.nextLine();

        // Variables de control para la edad
        int edad = 0;
        boolean edadValida = false;

        // 2. Pedir edad con control de errores en bucle
        while (!edadValida) {
            try {
                System.out.println("Escribe tu edad (debe ser un número entero):");
                // nextInt() lee el siguiente token como un número entero
                edad = sc.nextInt();
                edadValida = true; // Si llega aquí, la lectura fue correcta y rompe el bucle
            } catch (InputMismatchException e) {
                // Si el usuario introduce letras o decimales, se captura el error
                System.out.println("Error: La edad debe ser un número entero válido.");
                sc.next(); // IMPORTANTE: Limpia el token incorrecto del búfer para evitar bucle infinito
            }
        }

        // 3. Salida final de datos procesados
        System.out.println("¡Hola " + nombre + "!");
        System.out.println("Tienes " + edad + " años.");
        
        sc.close();
    }
}

Análisis Técnico del Código

  • sc.nextLine() vs sc.next(): El método next() detiene la lectura al encontrar el primer espacio en blanco (rompiendo nombres compuestos o apellidos). Usamos nextLine() porque absorbe todo el flujo de caracteres hasta que encuentra el salto de línea definitivo (\n).
  • InputMismatchException: Es una excepción lanzada automáticamente por Scanner cuando el elemento recuperado del flujo de entrada no coincide con los patrones del tipo de datos esperado (por ejemplo, introducir "veinte" cuando se invoca a nextInt()).
  • Limpieza del Búfer (sc.next()): Cuando nextInt() falla debido a una entrada incorrecta, el dato erróneo no se elimina de la memoria temporal. Si no se añade la instrucción de limpieza sc.next() dentro del bloque catch, el bucle volverá a leer el mismo dato incorrecto una y otra vez de forma indefinida, provocando un bucle infinito de errores.
