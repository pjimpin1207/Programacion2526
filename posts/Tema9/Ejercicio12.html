Ejercicio 12: Lectura, Escritura y Mutación de Caracteres con Acceso Aleatorio

Enunciado

En esta práctica avanzada consolidamos el uso de la clase RandomAccessFile combinando operaciones de control físico, saltos dinámicos en el buffer mediante punteros e inspección iterativa byte por byte.

Pasos requeridos:

  • Crear un fichero (controlando que si ya existiese de una ejecución previa, lo elimine por completo).
  • Abrir un flujo interactivo e introducir la cadena inicial "defg".
  • Añadir al principio del archivo la cadena "abc" y al final la cadena "hij".
  • Recorrer el fichero resultante para sustituir de manera quirúrgica todas las vocales por el carácter comodín "*".
---

Solución Completa

Hacemos uso del método delete() de la clase File para el reinicio del recurso, y de las propiedades de posicionamiento seek(0) y seek(random.length()) para las mutaciones dinámicas de la cadena.

package Tema9.Ejercicios.ejercicio12;

import java.io.File;
import java.io.IOException;
import java.io.RandomAccessFile;

public class Ejercicio12 {
    public static void main(String[] args) {
        File file = new File("datos.txt");

        try {
            // 1. Controlar la existencia y eliminar el fichero previo para limpiar la prueba
            if (file.exists()) {
                file.delete();
            }

            // 2. Acceso aleatorio inicial para introducir "defg"
            try (RandomAccessFile random = new RandomAccessFile(file, "rw")) {
                random.writeBytes("defg");
            }

            // Mostrar estado inicial
            System.out.print("Contenido inicial: ");
            imprimirFichero(file);

            // 3. Añadir "abc" al principio y "hij" al final del flujo
            try (RandomAccessFile random = new RandomAccessFile(file, "rw")) {
                // Posicionamos al principio (índice 0)
                random.seek(0); 
                random.writeBytes("abc"); // Sobrescribe los bytes correspondientes

                // Saltamos dinámicamente hasta el byte final actual del archivo
                random.seek(random.length());
                random.writeBytes("hij");
            }

            System.out.print("Contenido tras añadir: ");
            imprimirFichero(file);

            // 4. Algoritmo de filtrado: sustituir las vocales por "*"
            try (RandomAccessFile random = new RandomAccessFile(file, "rw")) {
                for (int i = 0; i < random.length(); i++) {
                    random.seek(i); // Movemos el puntero al byte actual
                    byte b = random.readByte(); // Lee el byte y avanza el puntero automáticamente
                    char c = (char) b;

                    // Si el carácter leído coincide con una vocal
                    if ("aeiouAEIOU".indexOf(c) != -1) {
                        random.seek(i); // Retrocedemos el puntero a la posición original para reescribir
                        random.writeBytes("*");
                    }
                }
            }

            System.out.print("Contenido final (sin vocales): ");
            imprimirFichero(file);

        } catch (IOException e) {
            System.err.println("Error de E/S: " + e.getMessage());
        }
    }

    // Método utilitario para volcar todo el contenido del fichero por consola de forma segura
    private static void imprimirFichero(File f) throws IOException {
        try (RandomAccessFile random = new RandomAccessFile(f, "r")) {
            random.seek(0);
            byte[] contenido = new byte[(int) random.length()];
            random.readFully(contenido);
            System.out.println(new String(contenido));
        }
    }
}

Análisis del Algoritmo de Sustitución

El comportamiento del puntero en operaciones de lectura/escritura:

Un error muy común al empezar a trabajar con RandomAccessFile dentro de bucles repetitivos es olvidar que las operaciones de lectura avanzan de manera automática el marcador.

  • Cuando el bucle hace random.readByte() en la posición i, recupera el byte correspondiente, pero el puntero interno del archivo se desplaza de forma inmediata a la posición i + 1.
  • Si descubrimos que el carácter leído es una vocal e invocamos directamente a writeBytes("*"), estaríamos escribiendo el asterisco en la posición siguiente, corrompiendo la estructura de la palabra.
  • La solución: Por eso es imperativo realizar un paso de reposicionamiento explícito invocando a random.seek(i) justo antes de ejecutar la escritura destructiva, obligando al marcador a retroceder a la celda de memoria que acabamos de validar.
