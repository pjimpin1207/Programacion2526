Ejercicio 7: Control y Metadatos de Archivos (Clase File)

Enunciado teórico e Investigación

El enunciado nos propone realizar una investigación profunda sobre la jerarquía de flujos en Java (paquete java.io), dividiéndose en dos grandes bloques según la naturaleza de los datos:

  • Flujos Orientados a Bytes (Streams): Clases como FileInputStream, FileOutputStream, BufferedInputStream, DataInputStream o PrintStream. Trabajan con datos binarios crudos de 8 bits (ideales para imágenes, audios o archivos ejecutables).
  • Flujos Orientados a Caracteres (Readers/Writers): Clases como FileReader, FileWriter, BufferedReader o PrintWriter. Manejan caracteres Unicode de 16 bits, adaptándose automáticamente al juego de caracteres (codificación) del sistema, optimizados para archivos de texto plano.

Antes de abrir cualquier flujo de lectura o escritura sobre un soporte físico, es una excelente práctica de desarrollo verificar el estado, existencia y permisos del archivo. Para ello, Java nos proporciona la clase File.

---

Solución Práctica

A continuación se presenta el código de la aplicación que inspecciona las propiedades y metadatos del archivo prueba.txt en el disco duro antes de lanzar operaciones de manipulación de flujos de datos.

package Tema9.Ejercicios.ejercicio7;

import java.io.File;

public class Ejercicio7 {
    public static void main(String[] args) {
        // 1. Definimos la ruta relativa del archivo objetivo
        File archivo = new File("prueba.txt");

        // 2. Comprobar si el archivo existe físicamente en el disco
        if (archivo.exists()) {
            System.out.println("--- Información del Archivo ---");

            // Nombre del fichero con su extensión
            System.out.println("Nombre: " + archivo.getName());

            // Ruta absoluta completa dentro del sistema operativo
            System.out.println("Ruta absoluta: " + archivo.getAbsolutePath());

            // Tamaño físico ocupado en el espacio de almacenamiento
            System.out.println("Tamaño: " + archivo.length() + " bytes");

            // Validación de los permisos de acceso de seguridad del sistema
            System.out.println("¿Se puede leer?: " + (archivo.canRead() ? "Sí" : "No"));
            System.out.println("¿Se puede escribir?: " + (archivo.canWrite() ? "Sí" : "No"));

            // 3. Verificar de manera condicional si es un fichero o un directorio
            if (archivo.isFile()) {
                System.out.println("Tipo: Es un fichero.");
            } else if (archivo.isDirectory()) {
                System.out.println("Tipo: Es un directorio.");
            }

        } else {
            System.out.println("El archivo no existe en la ruta especificada.");
        }
    }
}

Análisis de la Clase File

¿Qué representa realmente un objeto File?

Es muy común pensar que al instanciar new File("prueba.txt") estamos creando o cargando el contenido de un documento en memoria. Esto es una concepción errónea:

  • La clase File representa únicamente una ruta de acceso abstracta (un camino lógico o path) hacia un archivo o directorio en el disco duro del sistema, independientemente de si existe o no en la realidad.
  • Actúa como un objeto puente necesario para realizar consultas operativas básicas (metadatos) antes de inicializar flujos pesados de bytes como FileInputStream o caracteres como FileReader.
  • Métodos mark() y reset(): Mencionados en la investigación teórica, son característicos de flujos avanzados como BufferedReader. El método mark(readAheadLimit) guarda la posición actual del flujo de lectura, y reset() permite volver a dicha posición guardada para releer los datos, siempre que no hayamos superado el límite de bytes especificado de búfer en memoria.
