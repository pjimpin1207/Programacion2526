Ejercicio 6: El Adivino Digital

Enunciado

En esta divertida práctica de flujos de consola, implementamos un programa interactivo de "predicción" humorística. El objetivo es consolidar el uso combinado de lecturas de texto completo mediante tokens y tipos de datos numéricos enteros.

Misión:

  • El programa debe preguntar primero: "¿Cuál es tu nombre?" utilizando nextLine().
  • A continuación, preguntará: "¿Cuántos años crees que vivirás?" utilizando nextInt().
  • Finalmente, devolverá una respuesta profética calculando una fecha de fallecimiento basada en la expectativa sumándole 10 años adicionales.

Solución Completa

A continuación se detalla el código fuente de la clase. Observa cómo realizamos la operación aritmética directamente antes de mostrar la salida formateada por consola.

package Tema9.Ejercicios.ejercicio6;

import java.util.Scanner;

public class Ejercicio6 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 1. Preguntar y capturar el nombre
        System.out.println("¿Cuál es tu nombre?");
        String nombre = sc.nextLine();

        // 2. Preguntar y capturar la expectativa de años
        System.out.println("¿Cuántos años crees que vivirás?");
        int añosExpectativa = sc.nextInt();

        // 3. Lógica del "adivino" y salida final
        int añosMuerte = añosExpectativa + 10;
        System.out.println("Hola " + nombre + ", los astros dicen que morirás a los "
                + añosMuerte + " por culpa de un café frío.");

        sc.close();
    }
}

Análisis del Comportamiento de los Flujos

El orden de los factores sí altera el búfer:

En este ejercicio la estructura funciona de manera perfecta debido a que invocamos primero a nextLine() y luego a nextInt(). Sin embargo, hay que recordar una regla de oro en Java para futuros proyectos:

  • Si solicitáramos primero el dato numérico con nextInt() y justo después el texto con nextLine(), el programa "saltaría" la pregunta del nombre sin dejar escribir al usuario.
  • Eso ocurre porque nextInt() lee el número entero pero deja el salto de línea (\n) flotando en el búfer de entrada. Al invocar inmediatamente a nextLine(), este absorbe ese salto de línea residual interpretando que el usuario pulsó Enter con una cadena vacía.
  • Solución cuando eso pasa: Colocar un sc.nextLine() vacío para "limpiar" el búfer justo después de consumir cualquier dato numérico.
