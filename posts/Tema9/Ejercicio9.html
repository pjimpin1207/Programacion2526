Ejercicio 9: Ficheros y Directorios con java.io.File

Enunciado

En esta práctica ponemos a prueba los métodos de manipulación y creación dinámica de recursos del sistema de archivos mediante la clase java.io.File, integrando un bloque de control de excepciones estructural para flujos.

Requisitos y Objetivos:

  • Verificar si existe el archivo notas.txt en la raíz del proyecto.
  • Si el archivo ya existe, mostrar en consola su nombre y ruta absoluta de acceso.
  • Si el archivo no se encuentra, invocar el método de creación física en disco de forma automática.
  • Implementar un control robusto frente a errores capturando excepciones de tipo IOException.
---

Solución Completa

Hacemos uso de archivo.exists() para bifurcar la lógica del flujo y del método controlado createNewFile() envuelto en una estructura defensiva try-catch.

package Tema9.Ejercicios.ejercicio9;

import java.io.File;
import java.io.IOException;

public class Ejercicio9 {
    public static void main(String[] args) {
        // 1. Definir el objeto File apuntando al archivo 'notas.txt'
        File archivo = new File("notas.txt");
        
        // 2. Verificar de manera condicional si el recurso ya existe
        if (archivo.exists()) {
            System.out.println("El archivo ya existe.");
            System.out.println("Nombre: " + archivo.getName());
            System.out.println("Ruta absoluta: " + archivo.getAbsolutePath());
        } else {
            // 3. Si no existe, se inicia el protocolo de creación física
            System.out.println("El archivo no existe. Se va a crear...");
            try {
                // createNewFile() intenta crear el archivo y devuelve true si lo consigue
                if (archivo.createNewFile()) {
                    System.out.println("Archivo creado con éxito.");
                    System.out.println("Nombre: " + archivo.getName());
                    System.out.println("Ruta: " + archivo.getAbsolutePath());
                } else {
                    System.out.println("No se pudo crear el archivo.");
                }
            }
            // 4. Captura obligatoria de excepciones de entrada y salida (E/S)
            catch (IOException e) {
                System.err.println("Ocurrió un error de E/S al manejar el archivo: " + e.getMessage());
            }
        }
    }
}

Comprensión de Métodos de Manipulación

Operaciones críticas sobre el Sistema de Archivos:

  • archivo.createNewFile(): Este método interactúa directamente con el sistema operativo para reservar los bloques de espacio en el disco duro. Al requerir acceso físico externo, es una operación propensa a fallos (falta de permisos, disco lleno, rutas corruptas), por lo que Java nos obliga por contrato a comprobar la excepción mediante try-catch de manera explícita (Checked Exception).
  • System.err.println(): A diferencia de la salida estándar habitual (System.out), el uso de System.err desvía el flujo del mensaje hacia el canal de errores del sistema, lo que en muchos entornos de desarrollo provoca que el texto se renderice automáticamente en color rojo para facilitar el rastreo.
  • Gestión defensiva: El control de la persistencia de archivos evita que un error de hardware o una restricción de seguridad detenga la ejecución completa de nuestra aplicación de forma abrupta.
