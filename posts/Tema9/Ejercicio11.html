Ejercicio 11: Acceso Aleatorio (Modificación de Puntos Específicos)

Enunciado

A diferencia del acceso secuencial (donde debemos leer un archivo desde el principio hasta el final), el acceso aleatorio nos permite posicionarnos en cualquier byte del documento de forma instantánea para leer o escribir información.

Requisitos del ejercicio:

  • Crear un fichero llamado record.txt que contenga el texto inicial: Nivel:001.
  • Hacer un programa en Java que salte directamente a la posición del número y lo cambie por 005 sin alterar la palabra fija.
  • Pedir al usuario por consola qué nivel desea establecer y actualizar dicha posición de memoria directamente.
---

Solución Completa

Para lograrlo, utilizamos la clase de utilidad avanzada RandomAccessFile en modo de lectura/escritura ("rw"), manipulando el puntero del archivo mediante el método estructural seek().

package Tema9.Ejercicios.ejercicio11;

import java.io.File;
import java.io.IOException;
import java.io.RandomAccessFile;
import java.util.Scanner;

public class Ejercicio11 {
    public static void main(String[] args) {
        String nombreFichero = "record.txt";
        Scanner sc = new Scanner(System.in);

        try {
            // 1. Inicializamos el archivo de persistencia con el valor base Nivel:001
            archivoInicial(nombreFichero);

            // 2. Acceso directo para cambiar el nivel a 005 de forma estricta
            try (RandomAccessFile random = new RandomAccessFile(nombreFichero, "rw")) {
                // Saltamos las letras 'N','i','v','e','l',':' (índices del 0 al 5). Nos ponemos en el byte 6.
                random.seek(6);
                random.writeBytes("005");
                System.out.println("Nivel actualizado a 005");
            }

            // 3. Actualización dinámica interactiva por parte del usuario
            System.out.println("¿Qué nivel quieres ponerte?");
            String nuevoNivel = sc.next();

            try (RandomAccessFile random = new RandomAccessFile(nombreFichero, "rw")) {
                // Volvemos a saltar el prefijo estático de control "Nivel:"
                random.seek(6);
                random.writeBytes(nuevoNivel);
            }

            System.out.println("Archivo 'record.txt' actualizado correctamente.");

        } catch (IOException e) {
            System.err.println("Error de E/S: " + e.getMessage());
        } finally {
            sc.close();
        }
    }

    // Método de soporte para crear la estructura limpia del archivo inicial
    private static void archivoInicial(String ruta) throws IOException {
        File archivo = new File(ruta);
        try (java.io.FileWriter fw = new java.io.FileWriter(archivo)) {
            fw.write("Nivel:001");
        }
        System.out.println("Archivo inicial creado con Nivel:001");
    }
}

Análisis de la Mecánica de Acceso Aleatorio

¿Por qué es tan importante RandomAccessFile?

  • El puntero del archivo: Imagina que el archivo es un array gigante de bytes en el disco duro. Cuando abres un flujo normal de lectura, el puntero se sitúa en el índice 0 y avanza secuencialmente. RandomAccessFile se comporta como una estructura libre que permite desplazar ese puntero a la posición que tú quieras.
  • random.seek(long pos): Desplaza el puntero del archivo el número de bytes indicado desde el inicio del documento. En nuestra cadena "Nivel:001", los caracteres ocupan las siguientes posiciones:
    N i v e l : 0 0 1
    0 1 2 3 4 5 6 7 8
    Al hacer seek(6), colocamos el marcador justo al inicio de los datos numéricos, sobrescribiendo lo que haya a continuación sin mutar el prefijo.
  • Modo "rw": Es el parámetro por contrato que indica que abriremos el canal en modo de lectura (Read) y escritura (Write) simultánea.
