Ejercicio 3: Lectura interactiva de flujo de entrada de bajo nivel

Enunciado

En esta práctica dejamos de lado temporalmente la clase de utilidad Scanner para comprender el funcionamiento real del flujo de entrada estándar a nivel de bytes. El programa debe:

  • Permitir la introducción de una cantidad indeterminada de caracteres por consola.
  • Detectar el final de la inserción cuando el usuario pulse la tecla Intro.
  • Reutilizar ese sistema interactivo para solicitar el nombre al usuario y devolver un saludo personalizado ("Hola mundo").

Solución

Hacemos uso directo de System.in.read(), el cual captura el flujo carácter a carácter devolviendo su valor entero ASCII correspondiente. Evaluamos el final de la entrada buscando el salto de línea (\n).

package Tema9.Ejercicios.ejercicio3;

import java.io.IOException;

public class Ejercicio3 {
    public static void main(String[] args) throws IOException {
        // 1. Mostrar instrucciones y permitir la introducción de caracteres indeterminados
        System.out.println("Introduzca varios caracteres y después pulse intro para finalizar");

        // Bucle de lectura de bajo nivel que vacía el flujo hasta detectar el salto de línea
        int leido;
        while ((leido = System.in.read()) != '\n' && leido != -1) {
            // El cuerpo del bucle permanece vacío porque solo queremos procesar el salto de línea
        }

        // 2. Aplicar el mismo sistema interactivo para un "Hola Mundo" personalizado
        System.out.println("Introduzca su nombre:");

        // Contenedor temporal dinámico para reconstruir la cadena del nombre
        String nombre = "";
        int caracter;

        // Leemos carácter a carácter hasta presionar Intro
        while ((caracter = System.in.read()) != '\n' && caracter != -1) {
            // Descartamos el retorno de carro (ASCII 13) que añaden entornos como Windows (\r)
            if (caracter != 13) {
                nombre += (char) caracter; // Casting de entero a tipo char
            }
        }

        // Salida formateada con el resultado final reconstruido
        System.out.println("Hola " + nombre + "!");
    }
}

Análisis de Conceptos Fundamentales

  • System.in.read(): Es el método base de la clase InputStream. Lee el siguiente byte de datos del flujo de entrada. Si no hay datos disponibles, el hilo se bloquea en consola esperando al usuario. Devuelve un valor de -1 si se alcanza el final del flujo.
  • El problema de los saltos de línea combinados (\r\n): Los sistemas operativos como Windows codifican el salto de línea con dos caracteres consecutivos: Retorno de carro (\r, ASCII 13) seguido de Nueva Línea (\n, ASCII 10). La condicional if (caracter != 13) se encarga de ignorar el retorno de carro para que no rompa la codificación de la cadena de texto de salida.
  • Reconstrucción de datos: Al ser un flujo orientado a bytes, para recuperar texto legible es imperativo aplicar un moldeo o casting explícito de datos ((char) caracter) antes de concatenarlo en el objeto String.
