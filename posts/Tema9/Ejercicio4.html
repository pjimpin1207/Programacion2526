Ejercicio 4: Análisis de lectura en bloque (System.in.read)

Enunciado

A partir del fragmento de código proporcionado en la actividad, el objetivo es deducir y comprobar experimentalmente el comportamiento de la lectura de flujos cuando se utiliza un array de bytes como almacén intermedio o búfer de entrada.

Código Base Planteado:

public static void main(String args[]) {
    byte b[] = new byte[5];
    try {
        System.in.read(b);
    } catch (IOException ioe) {
        System.out.println(ioe);
    }
    String s = new String(b);
    System.out.println(s);
}

Solución Completa

A continuación se muestra el código listo para compilar, asegurando la importación de la excepción de entrada y salida (IOException) requerida para gestionar el bloque de control de errores.

package Tema9.Ejercicios.ejercicio4;

import java.io.IOException;

public class Ejercicio4 {
    public static void main(String args[]) {
        // 1. Declaramos un array de bytes de tamaño fijo (5 posiciones)
        byte b[] = new byte[5];
        
        try {
            // 2. Lee un bloque de bytes de la consola y los almacena en el array
            System.in.read(b);
        } catch (IOException ioe) {
            // Control de posibles excepciones de E/S
            System.out.println(ioe);
        }
        
        // 3. Convertimos el array de bytes resultante en una cadena de texto legible
        String s = new String(b);
        
        // 4. Imprimimos el resultado obtenido
        System.out.println(s);
    }
}

¿Qué hace exactamente este código?

Este programa lee una secuencia de datos por teclado de una sola vez, pero con un límite físico estricto determinado por el tamaño del array (en este caso, 5 bytes).

  • Si introduces menos de 5 caracteres (por ejemplo: "Hola" y pulsas Intro): El array capturará las letras 'H', 'o', 'l', 'a' junto con el carácter oculto del salto de línea (\n), rellenando las posiciones restantes del bloque de memoria de forma íntegra.
  • Si introduces exactamente 5 caracteres (por ejemplo: "12345"): El array se llenará por completo con dichos caracteres, dejando el salto de línea flotando en el flujo de entrada para la siguiente lectura.
  • Si introduces más de 5 caracteres (por ejemplo: "Estructura"): El método read(b) truncará la entrada. Solo absorberá los primeros 5 bytes ("Estru") y descartará o dejará en espera el resto de la palabra en el búfer del sistema.

Análisis de los Métodos Utilizados

  • System.in.read(byte[] b): A diferencia de su versión sobrecargada sin parámetros (que lee un único carácter a la vez), esta variante lee múltiples bytes del flujo de entrada y los vuelca directamente en el array proporcionado como argumento, devolviendo un entero con el número real de bytes leídos.
  • new String(byte[] bytes): Es un constructor especial de la clase String que decodifica los bytes de la memoria RAM utilizando el juego de caracteres por defecto del sistema operativo, transformando el array binario original en caracteres de texto legibles.
