Ejercicio 2

Enunciado

En este ejercicio analizamos el comportamiento interno de los flujos de salida mediante un experimento práctico controlado por tiempo utilizando la clase FileWriter.

Código Base Planteado:

import java.io.FileWriter;

public class ExperimentoFlush {
    public static void main(String[] args) throws Exception {
        // Creamos el escritor hacia un archivo
        FileWriter escritor = new FileWriter("prueba.txt");
        
        // Escribimos algo
        escritor.write("¿Donde está mi texto?");
        
        System.out.println("He escrito en el archivo... ¿o no?");
        
        // El programa se queda esperando aquí para que no termine
        Thread.sleep(10000); // Espera 10 segundos
        
        // No hemos puesto ni flush() ni close()
    }
}

Cuestionario de Observación

¿Se ha escrito algo en prueba.txt durante la ejecución? ¿Por qué?

No. Si abrimos el archivo prueba.txt mientras el programa está dormido en los 10 segundos de espera, veremos que está completamente vacío. Esto se debe a que Java no envía los caracteres directamente al disco físico con cada instrucción write(), sino que los almacena en la memoria temporal interna (buffer) del flujo para optimizar el rendimiento del sistema.

Solución

Para asegurar que los datos viajen de la memoria intermedia al disco duro de forma persistente, es obligatorio cerrar el flujo utilizando el método close() antes de pausar o finalizar el hilo de ejecución principal.

package Tema9.Ejercicios.ejercicio2;

import java.io.FileWriter;

public class Ejercicio2 {
    public static void main(String[] args) throws Exception {
        // Inicializamos el descriptor del archivo
        FileWriter escritor = new FileWriter("prueba.txt");

        // Escribimos el mensaje en la memoria temporal del buffer
        escritor.write("¿Donde está mi texto?");

        // SOLUCIÓN: Al cerrar el flujo, forzamos de manera automática el volcado de datos (flush)
        escritor.close();

        // El hilo entra en pausa, pero los datos ya se grabaron físicamente en el disco
        Thread.sleep(10000);
    }
}

Análisis de Conceptos Fundamentales

  • Mecanismo de búfer: El acceso a operaciones de E/S físicas en disco es lento. Java utiliza buffers en RAM para agrupar los datos y realizar una única escritura consolidada en bloque.
  • flush(): Método explícito para vaciar los datos acumulados en la RAM hacia el destino final sin dar por cerrado el canal.
  • close(): Realiza un volcado estructural completo y libera los recursos del archivo en el Sistema Operativo. Intentar realizar una escritura tras cerrar el archivo producirá una excepción de tipo IOException.
