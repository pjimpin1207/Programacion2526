Ejercicio 17: Serialización Colectiva — Exportación de Listas a Formato XML

Enunciado y Objetivo

En esta práctica subimos de nivel el uso de la API DOM. En lugar de generar etiquetas estáticas sueltas, aprenderemos a **mapear de forma dinámica una colección de datos de la memoria RAM (una lista de contactos)** para transformarla en un documento estructurado e intercambiable en formato XML.

Requisitos planteados:

  • Modelar una clase encapsulada Persona con los atributos básicos de localización: nombre, dirección y teléfono.
  • Instanciar en el método principal un ArrayList poblado con múltiples objetos de prueba.
  • Recorrer la lista de manera iterativa generando subárboles jerárquicos independientes para cada contacto y volcarlos al archivo físico contactos_.xml.
---

Estructura del Código

1. La Clase Modelo (Persona)

Mantenemos el contrato de encapsulamiento e incluimos el soporte para Serializable junto con una redefinición limpia de toString() para auditar la información por la consola del sistema.

package Tema9.Ejercicios.ejercicio17;

import java.io.Serializable;

public class Persona implements Serializable {
    private static final long serialVersionUID = 1L;

    private String nombre;
    private String direccion;
    private String telefono;

    public Persona(String nombre, String direccion, String telefono) {
        this.nombre = nombre;
        this.direccion = direccion;
        this.telefono = telefono;
    }

    public String getNombre() { return nombre; }
    public String getDireccion() { return direccion; }
    public String getTelefono() { return telefono; }

    @Override
    public String toString() {
        return "Persona{nombre='" + nombre + "', direccion='" + direccion + "', telefono='" + telefono + "'}";
    }
}

2. Programa de Mapeo Estructural (Ejercicio17)

Utilizamos un bucle for-each para procesar la lista. En cada iteración creamos un contenedor intermedio <persona>, le inyectamos los elementos hoja correspondientes con sus textos y los acoplamos al nodo raíz general.

package Tema9.Ejercicios.ejercicio17;

import org.w3c.dom.Document;
import org.w3c.dom.Element;
import javax.xml.parsers.DocumentBuilder;
import javax.xml.parsers.DocumentBuilderFactory;
import javax.xml.transform.OutputKeys;
import javax.xml.transform.Transformer;
import javax.xml.transform.TransformerFactory;
import javax.xml.transform.dom.DOMSource;
import javax.xml.transform.stream.StreamResult;
import java.io.File;
import java.util.ArrayList;
import java.util.List;

public class Ejercicio17 {
    public static void main(String[] args) {
        // 1. Instanciar la base de datos volátil en memoria RAM (Lista de Contactos)
        List<Persona> contactos = new ArrayList<>();
        contactos.add(new Persona("Ana García López", "Calle Mayor 12, 29001 Málaga", "600111222"));
        contactos.add(new Persona("Carlos Ruiz Pérez", "Avenida de Andalucía 45, 29004 Málaga", "611333444"));
        contactos.add(new Persona("María Torres Sánchez", "Plaza de la Constitución 3, 29015 Málaga", "622555666"));
        contactos.add(new Persona("Luis Fernández Mora", "Calle Larios 8, 29005 Málaga", "633777888"));
        contactos.add(new Persona("Laura Martínez Díaz", "Paseo del Parque 2, 29016 Málaga", "644999000"));

        try {
            // 2. Inicializar el lienzo del árbol DOM estructurado
            DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
            DocumentBuilder builder = factory.newDocumentBuilder();
            Document doc = builder.newDocument();

            // 3. Crear el nodo de control raíz principal 
            Element raiz = doc.createElement("contactos");
            doc.appendChild(raiz);

            // 4. Algoritmo de mapeo iterativo: Un nodo  por cada objeto en la lista
            for (Persona p : contactos) {
                // Creamos el contenedor de entidad para el registro actual
                Element persona = doc.createElement("persona");
                raiz.appendChild(persona);

                // Insertamos la propiedad encapsulada 
                Element nombre = doc.createElement("nombre");
                nombre.appendChild(doc.createTextNode(p.getNombre()));
                persona.appendChild(nombre);

                // Insertamos la propiedad encapsulada 
                Element direccion = doc.createElement("direccion");
                direccion.appendChild(doc.createTextNode(p.getDireccion()));
                persona.appendChild(direccion);

                // Insertamos la propiedad encapsulada 
                Element telefono = doc.createElement("telefono");
                telefono.appendChild(doc.createTextNode(p.getTelefono()));
                persona.appendChild(telefono);
            }
            
            // 5. Configurar el motor de transformación (Transformer) para el volcado físico
            TransformerFactory tf = TransformerFactory.newInstance();
            Transformer transformer = tf.newTransformer();

            // Directivas estéticas y de control de codificación universal
            transformer.setOutputProperty(OutputKeys.INDENT, "yes");
            transformer.setOutputProperty(OutputKeys.ENCODING, "UTF-8");
            transformer.setOutputProperty(OutputKeys.OMIT_XML_DECLARATION, "no");
            transformer.setOutputProperty("{http://xml.apache.org/xslt}indent-amount", "4");

            // Definimos el origen (nuestro árbol de memoria) y el destino (fichero contactos_.xml)
            DOMSource source = new DOMSource(doc);
            StreamResult result = new StreamResult(new File("contactos_.xml"));

            // Consolidación de flujos binarios hacia el disco duro
            transformer.transform(source, result);
            System.out.println("Fichero estructurado 'contactos_.xml' generado correctamente.");

            // 6. Auditoría visual de los datos procesados por consola
            System.out.println("\nResumen de contactos salvaguardados:");
            for (Persona p : contactos) {
                System.out.println("  • " + p);
            }

        } catch (Exception e) {
            System.err.println("Error al generar el documento XML: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

Análisis Arquitectónico de la Solución

Ventajas de la combinación de Listas con la API DOM:

  • Flexibilidad del Modelo de Nodos: Al acoplar las llamadas a createElement dentro de estructuras iterativas (como los bucles for), el tamaño de nuestro archivo resultante es completamente elástico. Funcionará exactamente igual si la lista contiene 5 contactos o si se procesa un lote masivo de miles de registros procedentes de una base de datos externa.
  • createTextNode frente a setTextContent: En este ejercicio hemos utilizado doc.createTextNode(valor) acoplado mediante appendChild. Aunque tiene un resultado idéntico a setTextContent() para valores simples de texto plano, la inserción de nodos de texto explícitos es más rigurosa desde el punto de vista arquitectónico y asegura que caracteres especiales (como los símbolos de las direcciones) se codifiquen de forma segura en las secuencias de escape nativas de XML (como &amp; o &lt;).
  • Interoperabilidad total: Al subir el archivo resultante a un repositorio compartido como GitHub, cualquier otra plataforma o lenguaje de programación (ya sea un script en Python, una aplicación web en JavaScript o un sistema nativo en C#) podrá interpretar la información sin pérdidas, ya que XML actúa como un estándar universal de transferencia de datos.
