Resumen de la Unidad 9: Entrada y Salida de Información

En esta unidad hemos estudiado cómo los programas en Java se comunican con el exterior (usuarios, discos duros, bases de datos o internet) utilizando un concepto fundamental de abstracción: los flujos de datos (Streams).

1. Naturaleza de los Flujos de Datos

Un flujo es una secuencia ordenada de datos que se transmite en serie desde un origen hacia un destino. Java los clasifica bajo diferentes criterios:

Criterio Tipos Principales
Según el Sentido Entrada (Input): Se lee información.
Salida (Output): Se escribe información.
Según el Tipo de Dato Bytes: Datos crudos o binarios (InputStream / OutputStream).
Caracteres: Texto decodificado en Unicode/ASCII (Reader / Writer).
Según el Acceso Secuencial: Se lee elemento a elemento desde el inicio.
Directo/Aleatorio: Permite saltar a una posición concreta.

2. Flujos Estándar y Scanner

Java mantiene abiertos algunos flujos predeterminados asociados a la consola del sistema operativo:

  • System.out y System.err: Flujos de salida estándar e impresión de errores.
  • System.in: Flujo de entrada estándar vinculado al teclado.

Para facilitar la lectura de la entrada estándar, usamos la clase Scanner, que proporciona una capa de abstracción para procesar datos primitivos y segmentar la información en "tokens" (palabras separadas por espacios).

3. Ficheros y Acceso Directo

Para lograr la persistencia de los datos, utilizamos archivos del sistema. La clase java.io.File representa la ruta o metadatos de un fichero o directorio, pero no lo abre ni lo crea directamente.

Acceso Aleatorio (RandomAccessFile)

A diferencia del acceso secuencial con FileReader o FileInputStream, la clase RandomAccessFile trata el archivo como un inmenso array de bytes. Posee un puntero interno que se puede desplazar a voluntad utilizando el método seek().

4. Serialización de Objetos

La serialización es el proceso mediante el cual convertimos el estado interno de un objeto (que vive en la memoria RAM volátil) en una secuencia de bytes para poder almacenarlo en un archivo o enviarlo por la red.

Reglas estrictas de Serialización:

  • La clase debe implementar la interfaz marcadora Serializable.
  • Se recomienda definir un identificador de versión serialVersionUID.
  • Si un atributo contiene datos sensibles (como contraseñas) o información que no debe guardarse, se le aplica el modificador transient. Los atributos estáticos tampoco se serializan.
  • Utilizamos ObjectOutputStream para escribir el objeto y ObjectInputStream para leerlo y reconstruirlo.

5. Procesamiento de XML con DOM

El XML es un lenguaje de marcado estandarizado que se utiliza para estructurar datos de forma independiente de la plataforma. Para manipularlo desde Java, la forma más habitual en archivos pequeños es utilizar la API DOM (Document Object Model).

DOM carga el documento XML completo en la memoria y lo convierte en un árbol jerárquico de nodos. Existen diferentes tipos de Nodos:

  • Document: La raíz conceptual del árbol completo.
  • Element: Representa una etiqueta XML completa (con apertura y cierre).
  • Text: El contenido textual que reside dentro de las etiquetas (o los saltos de línea y espacios en blanco).
