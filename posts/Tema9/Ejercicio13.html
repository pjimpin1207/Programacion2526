Ejercicio 13: Serialización y Deserialización de Objetos

Enunciado

La serialización es el proceso de transformar el estado de un objeto en una secuencia de bytes para poder almacenarlo en un archivo físico o transmitirlo a través de una red. La operación inversa, reconstruir el objeto original en memoria a partir de esos bytes, se denomina deserialización.

Misión planteada:

  • Crear una clase molde llamada Persona que sea apta para ser congelada en el disco.
  • Instanciar un objeto con el nombre de un personaje famoso (en nuestro caso, "Ronaldo") y guardarlo de forma binaria en un archivo persona.dat.
  • Implementar la lógica complementaria de lectura para deserializar el fichero, recuperando el objeto original y desvelando la identidad del personaje por consola.
---

Estructura del Código

1. La Clase de Datos (Persona)

Para que un objeto sea serializable en Java, su clase debe implementar obligatoriamente la interfaz marcadora Serializable. También es una buena práctica declarar un serialVersionUID para garantizar la compatibilidad durante la reconstrucción.

package Tema9.Ejercicios.ejercicio13;

import java.io.Serializable;

public class Persona implements Serializable {
    // Identificador único de versión de la clase para el proceso de deserialización
    private static final long serialVersionUID = 1L;

    private String nombre;

    public Persona(String nombre) {
        this.nombre = nombre;
    }

    public String getNombre() {
        return nombre;
    }
}

Aquí realizamos ambas fases. Primero abrimos un flujo de salida para guardar el objeto y, acto seguido, un flujo de entrada para leerlo y comprobar que la información se ha recuperado intacta en memoria.

            Persona personaDeserializada = (Persona) in.readObject();
            
            in.close();
            
            // Mostramos el resultado recuperado por pantalla
            System.out.println("¡Objeto deserializado con éxito!");
            System.out.println("Nombre desvelado: " + personaDeserializada.getNombre());

        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}

Análisis de los Mecanismos de Flujo Binario

¿Cómo interactúan estas clases especiales?

  • La interfaz Serializable: No contiene métodos que programar; actúa simplemente como una directiva o "bandera" para la Máquina Virtual de Java (JVM). Si intentamos pasar un objeto por writeObject() cuya clase no tiene este marcador, el programa lanzará inmediatamente una excepción NotSerializableException.
  • Flujos de bajo nivel vs. alto nivel:
    • FileOutputStream y FileInputStream son flujos orientados a bytes planos encargados de la conexión directa de tuberías con los archivos del disco duro.
    • ObjectOutputStream y ObjectInputStream son envoltorios avanzados de procesamiento que traducen la estructura lógica del grafo de objetos de la memoria RAM a formato binario y viceversa.
  • ClassNotFoundException: Al deserializar un archivo externo (por ejemplo, el que comparta un compañero), es indispensable que nuestro proyecto tenga acceso al archivo de la clase molde (Persona.class). Si la estructura del paquete o la clase no existe en el entorno receptor, el programa fallará al no saber cómo modelar los bytes leídos.
